### 09:48:22 - Response

Yes — by design. Two synchronization points enforce it:

**FSx itself is done before Phase 2 even starts.** FSx is created in **Phase 1** by `deploy-storage` (CloudFormation blocks until the volume reaches `CREATE_COMPLETE`, ~3-5 min). `init-fsx-layout` runs immediately after in the same sequential Phase 1, creating the layout directories. Phase 2 only starts after Phase 1 fully completes. So when Phase 2's parallel tracks kick off, FSx + its directory layout already exist.

(Side correction to your mental model: `deploy-compute` doesn't *start* FSx — it starts the EC2 instances that *mount* the already-existing FSx. FSx and the compute fleet are independent stacks.)

**Database and cache completion is enforced by `wait`.** In the Phase 2 shell block (`Makefile:556-577`):

```bash
$(MAKE) build-amis    ... & AMI_PID=$!
$(MAKE) deploy-database ... & DB_PID=$!
$(MAKE) deploy-cache  ... & CACHE_PID=$!
...
wait $CACHE_PID; CACHE_RC=$?
wait $DB_PID;    DB_RC=$?
wait $AMI_PID;   AMI_RC=$?

if [ $AMI_RC -ne 0 ] || [ $DB_RC -ne 0 ] || [ $CACHE_RC -ne 0 ]; then
    exit 1
fi
```

`wait <pid>` blocks the recipe until that PID exits. Three sequential `wait`s guarantee all three tracks finished. The return-code check at the end means **if ANY track failed, the recipe exits before Phase 3 runs.** Phase 3's compute deploy literally can't start until Phase 2's `wait`s all return AND the rc check passes.

So at the moment compute instances boot in Phase 3:
- ✓ FSx exists with `/fsx/nginx/sites-enabled/` (Phase 1)
- ✓ AMIs are AVAILABLE (Phase 2 AMI track)
- ✓ RDS is CREATE_COMPLETE and `/<env>/rds/endpoint` is in SSM (Phase 2 DB track — `deploy-database` blocks on stack-create-complete waiter)
- ✓ Valkey is up and `/<env>/cache/*` is in SSM (Phase 2 cache track — same)
- ✓ AMI IDs in SSM (Phase 3, first action before deploy-compute)

Every prereq the boot scripts touch is guaranteed in place. That's the whole point of the reorder — silly question is the kind worth asking before kicking off the destroy-deploy.
