## On the bash prompt env indicator (for later)

Great idea, very doable. Three options worth considering:

**Option A — Right-aligned prompt (closest to what you described, like zsh's `RPROMPT`):**

For zsh on the deploy-host:
```zsh
# ~/.zshrc or /etc/zsh/zshrc.d/worxco-env.zsh
RPROMPT='%F{yellow}[env:${WORXCO_ENV:-NONE}]%f'
```

For bash (a bit hackier — bash has no native right-prompt, but you can simulate with `\033[K` and cursor positioning):
```bash
# /etc/profile.d/worxco-env.sh
prompt_right() {
  printf "\033[%sG%s" "$(($(tput cols) - ${#1}))" "$1"
}
PROMPT_COMMAND='prompt_right "[env:${WORXCO_ENV:-NONE}]"'
```

**Option B — Color the LEFT prompt boldly:**

Less elegant but rock-solid: prepend `[ENV:sandbox]` in bright color to PS1. Works in any shell, any terminal width, copy-paste safe.

```bash
# /etc/profile.d/worxco-env.sh
PS1="\[\e[1;33m\][env:${WORXCO_ENV:-NONE}]\[\e[0m\] $PS1"
```

**Option C — Both:** colored left tag for redundancy + right-aligned for "always visible without scanning".

The `use-env` command writes `/etc/worxco/current-env`, every new shell sources it via `/etc/profile.d/worxco-env.sh`, prompt updates. Existing shells need a `re-load` (or close+reopen).

I'd suggest **Option A on zsh** since that's what most ops shells default to these days, plus the colored left tag as a fallback in case someone has a narrow terminal. The right-aligned version is genuinely "you cannot miss it" — sits at the edge of every prompt line.
