## Changes Made

### 1. `time-tracker.sh` (hook) — Cross-midnight support
- **Before**: Auto-closed sessions when calendar date changed at midnight
- **After**: Session boundaries based purely on idle timeout:
  - <30 min gap = same block (extend)
  - 30 min to 4 hrs = new block in same session (break, e.g., lunch)
  - 4+ hrs = new session (closes previous, starts fresh)
- Session date is always the date of the first block (11 PM to 2 AM = one session dated on the start day)
- Blocks now store `start_epoch` and `stop_epoch` for precise time-range queries

### 2. `prompt-logger.sh` — Git commit capture (both paths)
- **PostToolUse**: When Claude runs `git commit`, logs the commit immediately and updates `.last_commit_hash`
- **UserPromptSubmit**: Checks for new commits since `.last_commit_hash` — catches your manual commits on the next interaction
- Both use shared `log_git_commit()` and `update_last_commit()` functions
- PostToolUse matcher expanded from `Write|Edit` to `Write|Edit|Bash`

### 3. `tk-close` skill — Chronological interleaving
- Uses block `start_epoch`/`stop_epoch` for git log queries (not calendar dates)
- Builds a merged timeline per block: prompt log timestamps + git commit timestamps
- Generates bullets in chronological order — naturally interleaved, not separated by source
- Handles cross-midnight sessions correctly

### The "close the lid" scenario — resolved

Even if you close the lid after a manual commit:
1. Next session: time-tracker detects the gap, auto-closes the previous session
2. `/tk-close` runs for the old session, queries `git log` by **epoch range** (not calendar date)
3. Your commit falls within the block's epoch range — it gets picked up and interleaved correctly
4. Bullets land on the correct day's timesheet

You don't need to change your workflow. The system adapts to you.
