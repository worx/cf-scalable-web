### 11:33:03 - Response

Not a dumb question — important one.

**Answer: it's wired through SSM, so editing the JSON param will Just Work.**

The chain:

1. `cloudformation/parameters/app-drupal-sandbox.json` has `DrupalSiteName`
2. `cf-app-drupal.yaml:188-197` writes that into SSM as `/sandbox/drupal/site-name`
3. `smoke-test-drupal` (Makefile:2324) reads `/sandbox/drupal/site-name` from SSM at runtime, falls back to `drupal-$(ENV).test` only if SSM is empty
4. PHP-FPM boot (`image-builder/configs/configure-php.sh:93`) reads the same SSM param into `env[DRUPAL_SITE_NAME]` so settings.php's `trusted_host_patterns` accepts it

So your flow:

```bash
# Edit cloudformation/parameters/app-drupal-sandbox.json — set DrupalSiteName to sandbox.envs.zoning-info.com
make deploy-app-drupal ENV=sandbox     # rewrites SSM /sandbox/drupal/site-name
make deploy-allX ENV=sandbox           # rest of stack; PHP-FPM instances boot with the new value
make smoke-test-drupal ENV=sandbox     # auto-reads the new hostname from SSM
```

**One subtlety worth knowing:** `smoke-test-drupal` curls the **ALB by its `amazonaws.com` DNS** and forges the `Host:` header. It doesn't actually go through `sandbox.envs.zoning-info.com` in DNS. So the smoke-test will pass regardless of whether your Route 53 alias is right; it only confirms that PHP-FPM/Drupal will *accept* that Host header. To test the real end-to-end DNS path (alias → ALB → nginx → PHP-FPM → Drupal), curl the public URL directly:

```bash
curl -sI http://sandbox.envs.zoning-info.com/ | head -5
```

If that returns `HTTP/1.1 200 OK`, you've validated the whole chain — DNS, ALB routing, nginx Host-based vhost, Drupal trusted_host_patterns. Worth adding a `make smoke-test-public ENV=sandbox` that does exactly this once DNS is settled. Want me to add it, or wait until you've confirmed it works manually?
