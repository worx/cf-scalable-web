## Strategies, matched to pause length

### A — Active work session (next few hours, intermittent)
**Recommendation: keep everything up.**

Tearing down and rebuilding compute eats more time than the cost saved. Step 4 needs you to:
- Edit AMI build script
- Rebuild AMIs (~25 min)
- Redeploy cf-compute-php (~5 min)
- Test request flow
- Probably iterate 2-3 times

Constant churn in the compute layer would dominate your time. Cost in $ for one full work day idle ≈ $6-7. Worth it.

### B — Overnight / between sessions (8-16 hours)
**Recommendation: scale compute ASGs to 0 + stop deploy-host.**

EC2 instances stopped don't accrue hour charges (just EBS, which is cheap). FSx, ALB, NLB, RDS still cost — but EC2 ASG hours go to zero. Saves ~$30-50/mo if half the time.

We don't have a Make target for ASG-scale-to-zero yet. I can add one in 5 minutes:
- `make pause-compute ENV=sandbox` — sets ASG desired/min/max to 0/0/0
- `make resume-compute ENV=sandbox` — sets back to original (probably 1-2 instances)

### C — Several days idle
**Recommendation: destroy the compute layer, keep the data layer.**

```bash
make destroy-compute ENV=sandbox CONFIRMED=yes
# Saves: ALB + NLB + NGINX ASG + PHP ASG ≈ $70-110/mo
# Keeps: FSx + RDS + Valkey + deploy-host + peering
# Recreate when needed: make deploy-compute ENV=sandbox (~10 min)
```

Drupal install on FSx and database in RDS survive intact. Browser-test-via-drush-rs still works (those don't use ALB/NGINX/PHP-ASG).

### D — Dormant for weeks
**Recommendation: `make destroy-all ENV=sandbox CONFIRMED=yes`, keep deploy-host.**

Saves nearly everything. Drupal install is gone. When you come back: `make deploy-allX` + `make install-drupal`. ~75 min round trip, fully reproducible.

For very long pauses (months), even destroy the deploy-host. The whole project is git-tracked; bring it back from zero.
