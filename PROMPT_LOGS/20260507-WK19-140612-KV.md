**Message:**
```
feat(make): Add deploy-allX — full lifecycle including database and cache

deploy-all leaves database and cache as separate manual steps because
they are optional runtime dependencies. For the common case ('I want
the full stack working — set it and forget it'), running three commands
across 75 minutes of waiting is annoying.

deploy-allX = deploy-all + (deploy-database || deploy-cache in parallel)

The data-layer pair runs in parallel since they have no inter-dependency.
RDS Multi-AZ takes ~15 min, ElastiCache ~10 min — running them
simultaneously rather than sequentially saves ~10 minutes.

Output is captured per-stack to temp files and printed after both
finish, so the user sees both results regardless of which one had
problems. If either fails, deploy-allX fails (with both logs visible).

If running on the deploy-host (where refresh-env-config is in PATH),
the target also refreshes /etc/worxco/envs/<env> + /etc/hosts FSx
entries automatically at the end. Otherwise it prints a hint to run
the refresh manually.

Total wall time: ~65 min (was ~75-80 min sequential).
```
