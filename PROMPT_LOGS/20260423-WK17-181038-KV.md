### 18:10:49 - Response

Correct. SSM routes through the AWS API — it talks to the SSM service, which talks to the agent on the instance. No IP address, no port, no network path between you and the instance. The instance ID is just an identifier, not a network address.

That's why SSM works even with no inbound ports open and why it doesn't matter if the IP changes on stop/start. It also works across VPCs, private subnets, NAT — anywhere the agent can reach the SSM endpoints outbound on 443.
