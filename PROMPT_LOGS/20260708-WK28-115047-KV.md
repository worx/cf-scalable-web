### 11:51:04 - Response

One-word fix. The `sslmode=allow` change put us back where we started.

## What happened

In libpq semantics:
- `sslmode=require` — **mandate** SSL. Encrypt or fail.
- `sslmode=allow` — try non-SSL first, fall back to SSL only if server refuses.

With `allow`, pgloader tried unencrypted first. RDS said "no" (because `rds.force_ssl=1`), and pgloader reported the original error before ever attempting the SSL retry.

The combination we want:
- **`?sslmode=require`** in the URL → force encryption
- **`--no-ssl-cert-verification`** on the command line → skip CA trust check

Both together = "encrypted connection, don't verify server identity."
