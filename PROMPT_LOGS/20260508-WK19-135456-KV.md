feat(make): Add install-helpers target for fast iteration on deploy-host helpers

After a git pull on the deploy-host, the helper scripts in
scripts/deploy-host/ may have changed but the installed copies in
/usr/local/{bin,sbin}/ are still the originals from bootstrap.sh.
Until now, syncing required running 'sudo install -m 0755 ...' for
each helper individually — annoying friction during iteration.

make install-helpers runs the same set of 'install' commands that
bootstrap.sh runs at boot, but without the apt installs or service
restarts. After 'git pull', this is the one command needed to refresh
all six helper scripts plus the sudoers entry:

  info-env, show-env, psql-env, valkey-env  (read-only, /usr/local/bin)
  mount-env, refresh-env-config             (need sudo, /usr/local/sbin)
  worxco-refresh-env-config.sudoers         (NOPASSWD entry)

Output reminds the user to re-run 'sudo refresh-env-config <env>' if
the cached env file format changed (e.g., today's switch from
'KEY=value' to 'export KEY=value').

Iteration loop becomes:
  cd ~/projects/cf-scalable-web && git pull
  make install-helpers
  sudo refresh-env-config sandbox    # only when format changes
  # ...do the actual thing...

Help section updated.
