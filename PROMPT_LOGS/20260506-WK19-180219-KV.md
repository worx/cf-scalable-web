### 18:02:38 - Response

The instance survives. No reboot, no replacement.

What we changed is **output-only** — just the expression that computes the value of the `SecurityGroupId` stack output. Outputs aren't part of resource state, so:

- The EC2 instance: untouched, keeps running
- The security group: untouched, same `sg-xxxxxxxxx` ID
- Open SSM sessions: continue working
- IAM role, instance profile, secrets: untouched

When you run `make deploy-deploy-host`, CloudFormation will:
1. Detect the template hash changed
2. Compute a no-op changeset for resources (because nothing resource-level changed)
3. Re-evaluate the outputs
4. Report `UPDATE_COMPLETE` in maybe 30 seconds

**One small thing**: the `make deploy-deploy-host` target does pre-delete the `SSM-SessionManagerRunShell` document and recreates it. That's normal hygiene for that target and doesn't affect existing SSM sessions (active sessions cache the document at start time). New sessions will use the recreated document, which is identical content.

Safe to run while you're connected via SSM session. Your shell stays alive.
