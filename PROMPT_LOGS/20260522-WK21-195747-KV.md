I dug pretty deep and **can't find a smoking gun for the intermittency**, which is its own data point. What I checked:

- **No apt activity at 23:45 UTC** in `/var/log/dpkg.log` — wasn't a package-install race
- **unattended-upgrades ran at 19:26 UTC**, well before the failure, and installed nothing
- **php8.3-pgsql is `ii` (installed)** right now AND the .so file timestamp is from Mar 20 2026 (base AMI) — nothing relocated it
- **PHP 8.3.6, `php -m`** lists `pdo_pgsql` cleanly right now
- **No commits touched install-drupal.sh's check** between the failure and your re-run — identical code, identical deploy-host

What that leaves is a **transient process-level failure** at that specific moment. Top candidates, none provable from the logs we have:

1. **PHP CLI startup non-zero + pipefail trap.** `set -o pipefail` is in effect; if `php -m` exited non-zero for any reason (warning that promoted to error in some INI config, brief /tmp permission hiccup, etc.), the `|| fail` triggered without `php -m`'s actual error being visible.
2. **`grep -q` SIGPIPE race.** `grep -q` exits on first match → SIGPIPE to upstream `php`. PHP-CLI on Linux *usually* handles SIGPIPE gracefully (writes don't fail), but a buffered-stdout edge case could exit non-zero. Combined with `pipefail`, that trips the fail.
3. **Memory/OPcache transient.** Deploy-host is small; if PHP-CLI hit any startup issue (OPcache file lock, /tmp pressure), the process could exit non-zero without producing a visible reason.

**The honest diagnosis:** the check is fragile-by-design — it discards stderr, treats `php -m` non-zero as "extension missing," and gives an actionable-but-wrong error message. It worked once and failed once on the exact same configuration, which means the check itself is the bug, not the environment.

**One-line fix worth doing**: instrument the failure path so the *next* time this happens, we have evidence. Want me to make the check print `php -v`, `php -m`, and the actual `pdo_pgsql` lookup result on failure? That way the error message is diagnostic instead of misleading, and a future failure tells us which of the three theories is real.
