Operator (Kurt) flagged that the prod MySQL password contains `*` and

### 17:14:08 - Response

Pushed as `1a778fe`. **Short answer to your question:**

| Layer | Special-char treatment for your password |
|-------|------------------------------------------|
| `read -s -p` (input) | Literal — type the password exactly as it is, no escaping |
| `${PROD_PW}` in unquoted heredoc | Literal substitution — the `*` and `$$` write to the file verbatim |
| **pgloader URL** | **URL-encode** — `*` → `%2A`, `$` → `%24` |

**The encoded version for the URL** (one-liner using `jq`):

```bash
PROD_PW_ENC=$(printf '%s' "$PROD_PW" | jq -sRr @uri)
```

After that, use `${PROD_PW_ENC}` in the load file's MySQL URL — not `${PROD_PW}`.

**Same treatment for the sandbox PG password**, by the way — the value pulled from `/etc/worxco/envs/sandbox` could just as easily have URL-reserved chars (we generated it randomly in app-drupal stack), so the doc now encodes both. The pgloader load-file block looks like:

```bash
read -s -p "Prod MySQL password: " PROD_PW; echo
PROD_PW_ENC=$(printf '%s' "$PROD_PW" | jq -sRr @uri)
SANDBOX_PW_ENC=$(printf '%s' "$DRUPAL_DB_PASS" | jq -sRr @uri)

cat > /tmp/zinew.load <<EOF
LOAD DATABASE
  FROM      mysql://worxco:${PROD_PW_ENC}@127.0.0.1:3306/zinew
  INTO      postgresql://${DRUPAL_DB_USER}:${SANDBOX_PW_ENC}@${DRUPAL_DB_HOST}:${DRUPAL_DB_PORT}/${DRUPAL_DB_NAME}

WITH include drop, create tables, create indexes, reset sequences, foreign keys,
     downcase identifiers, batch rows = 1000, prefetch rows = 1000

CAST type tinyint when (= 1 precision) to boolean drop typemod using tinyint-to-boolean,
     type datetime to timestamptz drop default drop not null using zero-dates-to-null,
     type date drop not null drop default using zero-dates-to-null,
     type timestamp drop default drop not null using zero-dates-to-null
;
EOF
chmod 600 /tmp/zinew.load
unset PROD_PW
```

You can verify the encoding worked before running pgloader:

```bash
echo "Encoded prod: $PROD_PW_ENC"
# Should show your password with %2A in place of *, %24 in place of $
cat /tmp/zinew.load | head -3
# Confirm the URL has the encoded password embedded
```

Then `pgloader /tmp/zinew.load`. Standing by for what you find in the run output.
