feat(make): extend deploy-allX to install Drupal end-to-end (~75 min, zero → 200)

Adds 3 phases to deploy-allX so a single command takes a fresh AWS
account from nothing to a Drupal site serving HTTP 200:

  Phase 6: deploy-app-drupal (Secrets Manager + SSM params)
  Phase 7: install-drupal-remote (SSM dispatch to deploy-host)
  Phase 8: smoke-test-drupal (curl ALB, assert 200)

scripts/install-drupal-remote.sh (new) is the SSM orchestrator. The
deploy-host script it dispatches:

  1. git fetch + reset --hard origin/main (sync to latest code)
  2. Self-heal: if `use-env` is missing (pre-cutover deploy-host),
     re-runs bootstrap.sh — this also closes the pending P0 #3
     (deploy-host cutover) for free on the next deploy-allX run
  3. Unmounts any stale pre-cutover /var/www/<env> + strips the
     fstab entry (so re-mounting at /var/www works cleanly)
  4. refresh-env-config <env>  (re-reads FSx DNS after destroy/redeploy)
  5. use-env <env>             (mounts new FSx at /var/www)
  6. make install-drupal ENV=<env>  (composer create-project + drush
     site:install — does the actual codebase + DB schema build)

Idempotent: skips step 6 if /var/www/drupal/.installed already exists
on FSx (so re-running deploy-allX without destroy in between is safe).

Also adds:
  - make install-drupal-remote ENV=... (callable standalone)
  - make smoke-test-drupal ENV=... (fetches ALB DNS from CFN output +
    site-name from SSM, curls, asserts HTTP 200)

Help text updated to reflect new ~75 min runtime and new targets.
