## On the deploy-host — refresh the helper, mount fresh, then install

Since the live `mount-env` is the older version, let's refresh it before continuing:

```bash
cd ~/projects/cf-scalable-web && git pull

# Refresh helpers in /usr/local/{bin,sbin}
sudo install -m 0755 scripts/deploy-host/mount-env /usr/local/sbin/mount-env

# Re-run mount-env so the fstab entry gets written for the existing mount.
# The script is idempotent — if already mounted, it just reports that.
# To force fstab refresh on an already-mounted env, unmount first:
sudo umount /var/www/sandbox && sudo mount-env sandbox

# Verify fstab entry is now present
grep -A1 "worxco-mount-env" /etc/fstab

# Now run the install
make install-drupal ENV=sandbox
```

After this, you're set up for the entire stop/start + replace cycle to handle itself automatically going forward.
