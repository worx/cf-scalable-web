**Recommendation for your first cycle:** kick it off on the existing-state sandbox (you don't need to destroy-all first — `build-amis-if-needed` will see current AMIs and skip, so you'll exercise the orchestration on a faster cycle ~5-10 min vs ~30-35).

```bash
make deploy-allXX ENV=sandbox
```

Then for a real destroy-rebuild stress test, do `make destroy-all ENV=sandbox` → `make deploy-allXX ENV=sandbox` and time it. Expected: ~30-35 min total. If something breaks, the per-track logs make it easy to find what.

**Failure modes worth knowing in advance:**
- Heartbeat in some terminals (tmux occasionally) flickers — purely cosmetic.
- If you Ctrl-C, in-flight tracks complete on their own. Don't slam Ctrl-C twice unless you really need to bail (second one exits without waiting; can leave stacks in CFN-IN-PROGRESS).
- Validation gate runs once at the top — if you break a template, you'll see the failure in seconds.

Standing by for results. When this passes a few cycles, the natural follow-ups are per-pipeline RecipeVersion + optional php74 + parallel destroy.
