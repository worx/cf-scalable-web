### 15:11:21 - Response

That long single-line regex on line 31 was overcomplicated and brittle. The error means nginx tokenized the pattern wrong (probably because of the unquoted `|` chars and `#` chars together). The fix: break it up into simpler locations and quote the regex patterns. Replace the whole file with this cleaner version:

```bash
sudo tee /var/www/nginx/sites-enabled/drupal.conf > /dev/null <<'NGXEOF'
# SPDX-License-Identifier: GPL-2.0-or-later
# Copyright (C) 2026 The Worx Company
# Author: Kurt Vanderwater <kurt@worxco.net>
#
# Drupal 11 vhost — lives on FSx, shared across nginx fleet.

server {
  listen 80;
  server_name drupal-sandbox.test ~^.+\.elb\.amazonaws\.com$;

  root /var/www/drupal/web;
  index index.php index.html;

  set_real_ip_from 10.200.0.0/16;
  real_ip_header X-Forwarded-For;
  real_ip_recursive on;

  # Deny hidden files (but allow .well-known for ACME etc.)
  location ~ /\.(?!well-known) {
    deny all;
    log_not_found off;
    access_log off;
  }

  # Block direct access to Drupal source-code file types
  location ~* "\.(engine|inc|install|module|profile|po|sh|sql|theme|twig|tpl|xtmpl|yml)$" {
    deny all;
  }

  # Block composer/build metadata at root
  location ~* "/(composer\.(json|lock)|web\.config)$" {
    deny all;
  }

  # Image styles — generated on demand
  location ~ ^/sites/.*/files/styles/ {
    try_files $uri @rewrite;
  }

  # Private files via Drupal's /system/files route
  location ~ ^(/[a-z\-]+)?/system/files/ {
    try_files $uri /index.php?$query_string;
  }

  # Static assets — cache aggressively
  location ~* "\.(css|js|png|jpg|jpeg|gif|ico|svg|woff2?|ttf|otf|eot|webp)$" {
    expires max;
    access_log off;
    log_not_found off;
    try_files $uri @rewrite;
  }

  # Clean URLs
  location / {
    try_files $uri /index.php?$query_string;
  }

  location @rewrite {
    rewrite ^ /index.php;
  }

  # PHP via FastCGI — switch php83 -> php74 to exercise the 7.4 pool instead
  location ~ "\.php$|^/update\.php" {
    fastcgi_split_path_info ^(.+?\.php)(|/.*)$;
    try_files $fastcgi_script_name =404;

    include fastcgi_params;
    fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
    fastcgi_param PATH_INFO $fastcgi_path_info;
    fastcgi_param QUERY_STRING $query_string;
    fastcgi_param HTTP_PROXY "";

    fastcgi_pass php83;
    fastcgi_intercept_errors on;
    fastcgi_read_timeout 120s;
    fastcgi_buffers 16 16k;
    fastcgi_buffer_size 32k;
  }
}
NGXEOF
sudo chown root:root /var/www/nginx/sites-enabled/drupal.conf
sudo chmod 644 /var/www/nginx/sites-enabled/drupal.conf
```

Then test+reload on the live box:

```bash
sudo nginx -t && sudo systemctl start nginx
```

**Key changes vs. the previous file:**
- Split the one mega-regex into three small ones (hidden files, source extensions, composer/build).
- **Quoted every regex pattern that contains `|` or `?` or `#`** — nginx tokenization gets confused by unquoted regex with shell-metacharacter-like chars in it.
- Dropped the weird `^#.*#$` Emacs-backup pattern from the original — overkill for sandbox.
- Dropped the variant-of-extension suffixes (`~`, `.bak`, `.swp`, etc.) — same reasoning, and Drupal's own `.htaccess` covers those anyway if anyone serves the files directly.
