### 07:40:54 - Response

Congrats on the full clean deploy. 🎉

The make target is `make admin-login-url ENV=sandbox` — it SSM-dispatches `drush user:login` on the deploy-host, which spits back a one-time login URL that auto-authenticates you as user/1 (admin). No password retrieval needed.

```bash
make admin-login-url ENV=sandbox
```

You'll get a URL like `https://sandbox.envs.zoning-info.com/user/reset/1/<timestamp>/<hash>/login`. Paste into your browser, you're in. After login, go to `/user/1/edit` to set a real password if you want password-based logins as a fallback.

**Why it's hard to find in `make help`:** The help text lists it as `make admin-login-url ENV=...` but it's grouped near the other operational/Drupal targets, not under "Auth" or anything obvious. Probably worth a help-text tweak — both for "where the heck is that command" and for surfacing the other related ones (`admin-ssh-key-{add,remove,list,sync}` for operator SSH keys, the `init-secrets` / `list-secrets` for Secrets Manager browsing).
