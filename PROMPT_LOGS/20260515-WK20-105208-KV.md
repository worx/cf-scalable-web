### 10:52:23 - Response

`psql-env` needs the environment as an argument — `psql-env sandbox` not just `psql-env`. The heredoc never reached postgres, so `ALTER USER` didn't run, and the standalone `psql` test confirmed that.

Try this:

```bash
NEW_PASS=$(aws secretsmanager get-secret-value \
  --secret-id worxco/sandbox/drupal/db-password \
  --region us-east-1 --query SecretString --output text)
echo "Setting drupal_user password to: $NEW_PASS"

# Use -c with a single statement (cleanest)
psql-env sandbox -c "ALTER USER drupal_user WITH PASSWORD '$NEW_PASS';"

# Quick verification — \du equivalent
psql-env sandbox -c "SELECT rolname, rolcanlogin FROM pg_roles WHERE rolname='drupal_user';"

echo ""
echo "=== Now connect AS drupal_user with that password ==="
PGPASSWORD="$NEW_PASS" psql \
  "host=sandbox-postgres.c256s2w8gam2.us-east-1.rds.amazonaws.com port=5432 dbname=drupal user=drupal_user sslmode=require" \
  -c "SELECT current_user, current_database(), now();"

unset NEW_PASS
```

1. `ALTER ROLE` from the first `psql-env` call
2. A row showing `drupal_user | t` from the verification query
3. `current_user | current_database | now` row from the as-drupal_user connection
