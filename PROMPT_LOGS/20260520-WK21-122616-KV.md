fix(deploy-host): single MOTD print + admin-ssh-key resilience + tilde expansion

Three fixes bundled from the same testing session:

1) MOTD double-print
   Symptom: After the previous /etc/profile.d + /etc/zsh/zshrc.d MOTD
   fix, operators saw MOTD printed TWICE on every SSM login.

   Root cause: Ubuntu's default /etc/zsh/zprofile contains
     emulate sh -c 'source /etc/profile'
   which re-runs /etc/profile inside zsh (in sh-emulation) AFTER bash
   exec'd into zsh. The second /etc/profile run reaches the
   /etc/profile.d/*.sh loop (the bash-side run had already exec'd
   away before getting there), so our 00-worxco-motd.sh fires and
   prints MOTD. The MOTD_SHOWN export doesn't escape zsh's emulation
   scope, so when /etc/zsh/zshrc.d/00-motd.zsh runs moments later it
   sees MOTD_SHOWN empty and prints MOTD again.

   Fix: gate /etc/profile.d/00-worxco-motd.sh on ZSH_VERSION being
   unset. ZSH_VERSION is set whenever the shell is zsh — even inside
   sh-emulation — so this skips the profile.d print exactly in the
   case where zsh's own zshrc.d will handle it. The bash-only path
   (NO_AUTO_ZSH=1) still gets MOTD from profile.d.

2) admin-ssh-key add: reject DSA keys
   DSA (ssh-dss) was deprecated by OpenSSH in 2015 and modern
   clients (current macOS, Ubuntu 22.04+) refuse to load DSA private
   keys with "invalid format". The add command now rejects ssh-dss
   pubkeys upfront with a clear error pointing at `ssh-keygen -t
   ed25519 ...` as the replacement.

3) admin-ssh-key list: don't die on unfingerprintable keys
   set -euo pipefail + a failed `ssh-keygen -lf -` would kill the
   listing mid-loop (caller saw exit 255 and an empty table after
   the header). Now individual fingerprint failures fall back to
   "(unfingerprintable)" and the listing continues.

4) Makefile admin-ssh-key-add: expand ~ in FILE
   `make admin-ssh-key-add NAME=kurt FILE=~/.ssh/foo.pub` previously
   failed with "no such file: ~/.ssh/foo.pub" because Make passes
   the literal tilde and bash doesn't expand it inside double quotes
   in the recipe. Recipe now does `eval echo $(FILE)` to perform
   tilde expansion at Make time, so the natural shell-style
   syntax works.
