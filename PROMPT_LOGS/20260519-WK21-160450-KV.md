### 16:05:13 - Response

Quick on the .zshrc puzzle: the prompt config lives at the **system level** (`/etc/zsh/zshrc.d/worxco-prompt.zsh`), installed by `bootstrap.sh`. That's why no `~/.zshrc` is needed — the right-prompt loads automatically for every user. The "2 questions" you saw was zsh's `zsh-newuser-install` wizard (fires when no `~/.zshrc` exists), not something we control. Pick `q` to dismiss and the system-level config takes over. Working as designed — just a sharper edge than I'd prefer.

Now back to the diagnostic — go ahead and run the psql connect:

```bash
PASS=$(aws secretsmanager get-secret-value \
  --secret-id worxco/sandbox/drupal/db-password \
  --query SecretString --output text)
PGPASSWORD="$PASS" psql -h "$DRUPAL_DB_HOST" -U drupal_user -d drupal -c '\du drupal_user'
```

(You already have `$DRUPAL_DB_HOST` set since you `source`d `/etc/worxco/envs/sandbox` — or `use-env sandbox` did it for you.)

That one command tells us whether the bug is in ALTER ROLE / our password sync, or in drush's URL parsing.
