## If MySQL is running but sudo mysql wants a password

This is a "root@localhost changed away from auth_socket somehow" scenario. Recovery via safe mode:

```bash
# Stop MySQL
sudo systemctl stop mysql

# Start with grant tables skipped (no auth required)
sudo mkdir -p /var/run/mysqld && sudo chown mysql:mysql /var/run/mysqld
sudo mysqld_safe --skip-grant-tables --skip-networking &

# Wait a couple seconds for it to come up, then connect and fix root
sleep 3
sudo mysql <<'SQL'
FLUSH PRIVILEGES;
ALTER USER 'root'@'localhost' IDENTIFIED WITH auth_socket;
SQL

# Kill the safe-mode instance and restart normally
sudo pkill -f mysqld_safe
sudo pkill -f mysqld
sleep 2
sudo systemctl start mysql

# Verify sudo mysql works again
sudo mysql -e "SELECT current_user();"
```
