For zsh on the deploy-host, there are two parts: **persistent** (committed, so future deploy-host rebuilds have it) and **immediate** (you can use it now, before rebuild).

## Persistent (commit to draft)

**1. `scripts/deploy-host/bootstrap.sh`** — add a "Install zsh + global config" step. This installs zsh, changes ubuntu's login shell, and drops a global zshrc with the RPROMPT.

**2. `cloudformation/cf-deploy-host.yaml`** — change the SSM-SessionManagerRunShell document's `exec bash -l` to `exec zsh -l`. **Important caveat**: that document is **account-level** (it affects every SSM session in the AWS account, per the existing gotchas.md note). If any other box you SSM into doesn't have zsh, those sessions will fail. Two safer paths:

   - **(a) safer:** keep the SSM doc invoking bash, but at the end of `/etc/bash.bashrc` (system-wide) on the deploy-host *only*, add `[ -t 0 ] && [ -x /usr/bin/zsh ] && exec zsh`. Bash starts (SSM happy), immediately execs into zsh.
   - **(b) cleanest:** change the SSM doc AND install zsh on every other box that uses SSM. Bigger commitment but more uniform.

I'd recommend (a). It's local to the deploy-host AMI, doesn't touch shared infrastructure, and degrades gracefully.

**3. `/etc/zsh/zshrc.d/worxco-prompt.zsh`** (also installed by bootstrap.sh):

```zsh
# Show the current Worxco environment on the right side of every prompt.
# The file /etc/worxco/current-env is written by `use-env <name>`.
autoload -U colors && colors

_worxco_env() {
  if [[ -r /etc/worxco/current-env ]]; then
    echo "[env:$(< /etc/worxco/current-env)]"
  else
    echo "[env:NONE]"
  fi
}

RPROMPT='%F{yellow}$(_worxco_env)%f'

# History
HISTFILE=~/.zsh_history
HISTSIZE=10000
SAVEHIST=10000
setopt INC_APPEND_HISTORY SHARE_HISTORY
```

## Immediate (paste into your current SSM session)

This works *right now*, doesn't survive a deploy-host rebuild — but lets you live in zsh today:

```bash
# 1. Install zsh
sudo apt-get update -qq
sudo apt-get install -y zsh

# 2. Make zsh ubuntu's login shell (persists in /etc/passwd until deploy-host rebuild)
sudo chsh -s /bin/zsh ubuntu

# 3. Drop a minimal zshrc with the right-prompt env indicator
cat > ~/.zshrc <<'EOF'
autoload -U colors && colors

_worxco_env() {
  if [[ -r /etc/worxco/current-env ]]; then
    echo "[env:$(< /etc/worxco/current-env)]"
  else
    echo "[env:NONE]"
  fi
}

PROMPT='%n@%m:%~%# '
RPROMPT='%F{yellow}$(_worxco_env)%f'

HISTFILE=~/.zsh_history
HISTSIZE=10000
SAVEHIST=10000
setopt INC_APPEND_HISTORY SHARE_HISTORY
EOF

# 4. Bootstrap current-env to sandbox so the prompt shows something
sudo mkdir -p /etc/worxco
echo sandbox | sudo tee /etc/worxco/current-env > /dev/null

# 5. Drop into zsh — RPROMPT lights up immediately
exec zsh -l
```

After step 5, your prompt should read:
```
ubuntu@ip-…:~%                                                    [env:sandbox]
```

Right-aligned, yellow. Unmissable.
