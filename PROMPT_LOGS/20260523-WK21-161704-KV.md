**Two architectural decisions I'd make unless you object:**

1. **Per-env cert, not wildcard.** Cert lives in `cf-compute-alb.yaml`, gets created with the stack, destroyed with the stack. Matches the "each env is independent" pattern. ACM certs are free, and AWS auto-renews. Cost: ~5 min for DNS-01 validation on each deploy. Wildcard would be cleaner across many envs but requires either a shared stack or hardcoding `*.envs.zoning-info.com` — anti-portability.

2. **HTTP → HTTPS 301 redirect on port 80.** Standard practice. Bookmarks to `http://` URLs still work. No reason to keep serving HTTP plaintext once HTTPS is available.
