fix(deploy-host): Export GroupId not GroupName from SecurityGroupId output

The SecurityGroupId output was using !Ref DeployHostSecurityGroup, which
returned the GroupName ("cf-deploy-host-sg") instead of the GroupId
("sg-xxxxxxxxx"). This is a legacy EC2-Classic Ref behavior that AWS
falls back to when a security group has no explicit VpcId — even though
EC2-Classic itself was retired in 2022 and the SG actually lives in the
default VPC.

The wrong export caused cf-deploy-peering to fail with:
  "Invalid id: cf-deploy-host-sg (expecting sg-...)"

Fix: use !GetAtt DeployHostSecurityGroup.GroupId, which always returns
the sg-xxxxxxxxx ID regardless of how the SG was created. Output-only
change — the SG resource itself is unchanged, so re-deploying does not
disturb the running deploy host.

After this fix, recovery sequence:
  1. make destroy-peering ENV=sandbox  (clean failed stack)
  2. git pull                           (get this fix)
  3. make deploy-deploy-host            (refresh export with correct value)
  4. make deploy-peering ENV=sandbox    (now succeeds)
