## What's happening

`QMYND:MYSQL-UNSUPPORTED-AUTHENTICATION` is pgloader's Common Lisp MySQL driver (called `qmynd`) failing because **MySQL 8 defaults to `caching_sha2_password` for authentication**, and `qmynd` only supports the older `mysql_native_password` plugin.

This is a very well-known pgloader-vs-MySQL-8 compatibility issue. When you created `worxco@localhost` during the local MySQL setup, MySQL 8 defaulted the new user to `caching_sha2_password`. pgloader can't handle that.

## Fix — switch the local MySQL user to mysql_native_password

```bash
sudo mysql <<'SQL'
ALTER USER 'worxco'@'localhost'
  IDENTIFIED WITH mysql_native_password
  BY 'localscratchpass';
FLUSH PRIVILEGES;
SQL
```

Verify the change:

```bash
sudo mysql -e "SELECT user, host, plugin FROM mysql.user WHERE user='worxco';"
```

You should see `mysql_native_password` in the `plugin` column.

Then rerun:

```bash
sudo rm -f /tmp/zinew-pgloader.log
sudo ./scripts/tt.sh
cat /tmp/zinew-pgloader.log
```
