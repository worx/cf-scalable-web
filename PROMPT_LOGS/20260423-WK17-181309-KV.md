### 18:13:20 - Response

Close — but a terminated instance is gone forever (disk and all). You'd want to **stop** it, not terminate it. A stopped instance:
- No compute cost
- No public IP cost (it releases the IP)
- Only EBS storage cost (~$1.60/month for 20GB gp3)
- Starts back up in about 30 seconds when you need it

The auto-stop is easy. A simple cron job on the instance itself:

```bash
# If no SSH/SSM sessions for 30 minutes, stop myself
if [ $(who | wc -l) -eq 0 ]; then
  aws ec2 stop-instances --instance-ids $(curl -s http://169.254.169.254/latest/meta-data/instance-id)
fi
```

Or even simpler — a CloudWatch alarm that watches `CPUUtilization < 5%` for 30 minutes and triggers an EC2 stop action. No agent needed, fully managed.

Then when you need it: `make start-deploy-host`, wait 30 seconds, SSM in.

Want me to add the CloudWatch auto-stop to the deploy host template? It's a few lines of CloudFormation — a metric alarm plus a stop action. No cron, no scripts, purely AWS-managed.
