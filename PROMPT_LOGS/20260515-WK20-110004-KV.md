### 11:08:46 - Git Commit

**Commit:** `26c4bde3c580`

**Message:**
```
fix(php-fpm): remove invalid `listen.allowed_clients = any` from www.conf

The 8.3 and 7.4 pool configs both had `listen.allowed_clients = any`,
which is NOT a valid value. The directive takes a comma-separated IP
allow-list; if left blank, all IPs are allowed (the default). Setting
it to the literal string `any` makes PHP-FPM treat that as "allow only
the IP literally named 'any'", which never matches anything, so every
connection is rejected with `ERROR: Connection disallowed: IP <addr>`
written to stderr (not the FPM error log).

nginx sees that rejection as "premature close" or "connection reset",
which is indistinguishable from a worker crash, a network problem, or
a bad password. Took multiple days to find — only strace on a worker
showed the actual rejection happening BEFORE any FastCGI parsing.

Comment the directive out (with a long note explaining why) instead of
just deleting it, so future readers don't reintroduce the bug.

Network-level ACL is enforced by the PHP security group, which only
allows the nginx SG on port 9000 — so removing this pool-level filter
doesn't widen the attack surface.
```
