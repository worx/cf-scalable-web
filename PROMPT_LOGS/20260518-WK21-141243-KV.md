## What changed

Two additions to `.obsidian/app.json`:

**`showUnsupportedFiles: false`** — explicit (it's the default but worth being explicit). Hides `.yaml`, `.sh`, `.py`, `.json`, etc. from the file explorer.

**`userIgnoreFilters: [...]`** — hides whole top-level directories that contain no notes-worthy markdown. After reload, your Obsidian file tree will show only:

```
notes/                  (auto-created on first new note)
assets/                 (auto-created on first attachment)
docs/                   (12 .md files — the real documentation)
README.md, TODO.md, etc. (root markdown)
.obsidian/              (dot-dir, hidden by default)
```

Everything in `cloudformation/`, `scripts/`, `image-builder/`, `deploy-host/`, `tests/`, `obsidian-minimal-template/`, and `PROMPT_LOGS/` is gone from view. The dotfile dirs (`.git`, `.claude`, `.idea`, `.qodo`) were already hidden by Obsidian's default behavior — no entries needed.
