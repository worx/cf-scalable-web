## The path is: React2 → jumpbox → S3 → deploy-host → FSx

We can't rsync directly from deploy-host to React2 — they're in different AWS accounts with no direct network path. But we already have the jumpbox in prod with SSH access to React2 (via the WebSSH SG allowing traffic from 172.31.16.0/20) and S3 write access to the sandbox migration bucket. So we use the jumpbox as a courier.

Two transfer streams because they have very different sizes and treatment:

1. **Codebase** (modules, themes, composer.json, vendor/, etc.) — small-to-medium, we exclude settings.php + files/
