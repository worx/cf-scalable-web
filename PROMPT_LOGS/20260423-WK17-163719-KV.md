### 16:37:27 - Response

No — `deploy` does a CloudFormation **update**, not a replace. If the stack already exists, CloudFormation compares the new template to the current state and only changes what's different. Since UserData changes don't force an instance replacement in CloudFormation (UserData only runs on first boot), the instance would stay as-is with the old broken config.

You need destroy then deploy to get a fresh instance that runs the new UserData:

```
make destroy-deploy-host && make deploy-deploy-host
```

This is one of the quirks of EC2 UserData — it's a one-shot bootstrap script, not a configuration management tool. If you need to change what's on the box, you rebuild the box.
