### 11:42:42 - Response

Good — progress. We got past the "no encryption" wall. Now SSL is negotiating, but pgloader is refusing to trust RDS's certificate chain.

## What the error means

RDS PostgreSQL uses AWS's own root CA (currently `rds-ca-rsa2048-g1` or similar) to sign its server cert. That CA is **not in the system trust store** on your deploy-host, so OpenSSL walks the chain, hits AWS's root CA, sees it's not in its trusted roots, and treats it as "self-signed" (which is misleading — it's not actually self-signed, just signed by an unknown-to-you CA).

pgloader's own error output points to the answer: **`You may try --no-ssl-cert-verification`**.

## Two options

| Option | Effort | Trust model |
|--------|--------|-------------|
| **Quick: skip verification** | 1 line | Encrypts the connection but doesn't verify the server is who it claims to be |
| **Proper: install RDS CA bundle** | ~3 commands | Full TLS with cert verification against AWS's published CA |

## Recommendation for right now

**Use `--no-ssl-cert-verification`** for this one-shot migration. Here's the reasoning:

- The connection is deploy-host → sandbox RDS, **entirely inside your sandbox VPC's private network**. There's no path where a MITM could sit.
- TLS is still established — the connection is still encrypted end-to-end. You're just not verifying the server's identity.
- The threat model for MITM inside an AWS VPC is essentially "someone has compromised AWS's own networking hardware." Not a realistic concern for a migration test.

## The edit to `scripts/tt.sh`

Change this line:

```bash
pgloader /tmp/zinew.load 2>&1 | tee /tmp/zinew-pgloader.log
```

to this:

```bash
pgloader --no-ssl-cert-verification /tmp/zinew.load 2>&1 | tee /tmp/zinew-pgloader.log
```

## For the long-term (post-migration cleanup)

Later, when you want to codify this properly in the runbook, the "install RDS CA" path is:

```bash
sudo curl -sSL \
  -o /usr/local/share/ca-certificates/aws-rds-global-bundle.crt \
  https://truststore.pki.rds.amazonaws.com/global/global-bundle.pem
sudo update-ca-certificates
```

Then pgloader without `--no-ssl-cert-verification` will work because OpenSSL's system trust store now includes AWS's RDS CA chain.

We can bake this into `docs/MIGRATION.md` and/or a deploy-host provisioning script once we prove out the full pgloader flow — no need to slow you down with it right now.
