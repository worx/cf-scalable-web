Done. New deploy-hosts will get `/etc/tmux.conf` automatically on bootstrap.

**For the host you're sitting on right now**, paste this into your deploy-host shell to install it without waiting for an AMI rebuild:

```bash
sudo tee /etc/tmux.conf > /dev/null <<'EOF'
set -g mouse on
set -g history-limit 50000
set -g mode-keys vi
set -sg escape-time 10
set -g base-index 1
setw -g pane-base-index 1
set -g status-interval 5
set -g status-right-length 60
set -g status-right '#[fg=yellow][env:#(cat /etc/worxco/current-env 2>/dev/null || echo NONE)]#[default] %H:%M'
bind r source-file /etc/tmux.conf \; display-message "tmux.conf reloaded"
EOF

# If you're already inside a tmux session, reload without re-attaching:
tmux source-file /etc/tmux.conf
```

**What you get:**
- `Ctrl-b [` then **`Ctrl-u`/`Ctrl-d`** half-page, **`Ctrl-b`/`Ctrl-f`** full-page, **`/`** forward search, **`?`** reverse, **`n`/`N`** next/prev match, **`g`/`G`** top/bottom, **`q`** quit
- **Trackpad scroll** works directly (no need to enter copy-mode)
- **50k lines** of scrollback so a full `make deploy-allX` stays in buffer
- **Status bar** shows the current env on the right (e.g. `[env:sandbox] 14:32`) — visible even when you're inside a long-running pane like `make serve-drupal-local` where the shell prompt isn't on screen
- **`Ctrl-b r`** reloads the config (useful while you're tweaking it)

Heads up: window/pane indexing flipped from 0-based to 1-based. If you have a habit of `Ctrl-b 0` to jump to first window, it's now `Ctrl-b 1`.
