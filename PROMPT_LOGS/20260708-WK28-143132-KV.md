## What went wrong

By wiping `/etc/mysql` **before** the MariaDB install, apt's `mariadb-server` postinst script didn't get to recreate the config-directory structure cleanly. MariaDB expects `/etc/mysql/mariadb.conf.d/` and `/etc/mysql/conf.d/` to be there, and now something's missing.

## Clean fix — purge and reinstall

```bash
# Stop MariaDB if it's running
sudo systemctl stop mariadb 2>/dev/null
sudo systemctl stop mysql 2>/dev/null

# Purge everything MariaDB-related (including config-fragment packages)
sudo apt purge -y mariadb-server mariadb-server-core mariadb-client mariadb-client-core mariadb-common

# Also purge any leftover mysql-* just in case
sudo apt purge -y 'mysql-*' 2>/dev/null

# Clean up leftover dirs
sudo rm -rf /etc/mysql /var/lib/mysql /var/log/mysql

# apt housekeeping to remove any orphaned config packages
sudo apt autoremove -y
sudo apt update

# Fresh install — postinst can now build the full config tree from scratch
sudo apt install -y mariadb-server mariadb-client
```

## Verify the structure this time

```bash
# Directory layout should now be complete
sudo ls -la /etc/mysql/
sudo ls -la /etc/mysql/mariadb.conf.d/
sudo ls -la /etc/mysql/conf.d/

# Service should be active
sudo systemctl status mariadb --no-pager

# Verify version and socket auth works
sudo mysql -e "SELECT VERSION(), current_user();"
```

You should see the config dirs populated, `mariadb` active, and the version query returning something like `10.11.x-MariaDB` with `current_user()` = `root@localhost`.
