### 15:37:31 - Response

Pushed as `22f43bc`. **The new public surface is much cleaner:**

```
make deploy-all                    Parallel full-stack deploy (default; ~30 min)
make deploy-all TYPE=linear        Sequential full-stack (~45-50 min; old behavior)
make deploy-all TYPE=no-data       Foundation + compute only — no RDS / cache / app-drupal
make destroy-all                   Parallel destroy (default; ~15-20 min)
make destroy-all TYPE=linear       Sequential destroy (~25 min)
```

Plus the install workflow stays the same separate target:
```
make install-drupal-full ENV=sandbox
```

**Migration story for anyone with muscle memory of the old names:**

- `make deploy-allXX` → still works, aliased to `deploy-all-parallel`
- `make deploy-allX` → still works, aliased to `deploy-all-linear`
- `make destroy-allXX` → still works, aliased to `destroy-all-parallel`
- `make deploy-all` (the OLD meaning, no-data) → **behavior changed.** Now defaults to the parallel full-stack. Anyone wanting the old no-data behavior must explicitly say `TYPE=no-data`. This is intentional — most operators typing `make deploy-all` want a complete env.

The internal canonical targets (`deploy-all-parallel`, `deploy-all-linear`, `deploy-all-no-data`, `destroy-all-parallel`, `destroy-all-linear`) are also callable directly if anyone prefers explicit naming over the dispatcher.

**Where you sit now:**

The infrastructure orchestration layer is in a really good shape. Operator surface is consolidated, three deploy options + two destroy options, all behind clean names. Past timings:

- `deploy-all` (parallel): 30m58s
- `destroy-all` (parallel): 17m23s
- `install-drupal-full`: ~5m

End-to-end destroy → deploy → install → working HTTPS: ~53 minutes. Down from the v0.2.0-https baseline of ~70-80 minutes.

This is a natural tag-able point — `v0.3.0-parallel-orchestration` or similar — if you want to lock this milestone in. Otherwise, what's next? Some candidates:

- **Per-pipeline RecipeVersion** (skip rebuilds when only one recipe changed) — would save ~15 min when iterating on a single AMI recipe.
- **Optional php74** parameter — skip the php74 ASG (and AMI bake) on envs that don't need it. Drupal 11 doesn't.
- **P1 architectural item:** salt-persistence to SSM SecureString (the production blocker).
- **P0 conversation:** the multi-site / multi-RDS architectural review.

Or just take a breather — this is a solid milestone.
