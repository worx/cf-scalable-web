### 15:34:37 - Response

These are **shell environment snapshots** created by Claude Code itself. Here's what they are:

**What**: A complete dump of your zsh session — every function definition, every alias, and your full `$PATH`. It's Claude Code capturing the state of your shell so that when it runs Bash commands, it can replicate your environment (your aliases, your PATH, your custom functions like `mp`, `mpl`, `wttr`, etc.).

**Created by**: Claude Code CLI. Each time you start a new Claude Code session (or when your shell environment changes significantly), it takes a snapshot. The filename encodes `zsh-{epoch}-{random}.sh`.

**How long to keep**: You don't need any of them. Claude Code creates a fresh one at session start. The oldest ones here are from November 2025 — completely stale. Only the most recent one (April 25) has any relevance, and even that gets replaced next session.

**Version-dependent?**: Not exactly tied to Claude Code version, but tied to your *shell configuration*. If you change your `.zshrc`, aliases, or PATH, the next session creates a new snapshot that supersedes all prior ones. The old ones are never read again.

**Bottom line**: 1 MB of dead weight. The `.gitignore` already excludes `shell-snapshots/`. You could `rm -rf ~/.claude/shell-snapshots/*` and Claude Code would regenerate a fresh one on your next session without blinking.

Same story as telemetry — runtime ephemera that Claude Code generates and forgets about. Not yours to maintain.
