Not a dumb question. Yes, technically — but the deploy host is in the default VPC and FSx is in the project VPC. They can't talk to each other without VPC peering, which we haven't set up.

The easier path for getting files onto FSx: use a PHP or NGINX instance that already has it mounted as a relay. Upload to S3 from the deploy host, then copy from S3 to `/var/www` on the PHP instance.

Or — do the Composer install and Drupal setup directly on the PHP instance via SSM. You're already SSM'd in. Once the S3 policy fix deploys, grab Composer and go.
