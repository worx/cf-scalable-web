### 13:23:01 - Git Commit

**Commit:** `803eaa2049d5`

**Message:**
```
perf(make): validate once per compound deploy + compress output to 1 line

Two related changes that together reclaim ~150 lines of screen during a
deploy-allX run:

1) Validate-once guard. The `validate` target now short-circuits to
   `exit 0` (silently) if the env var VALIDATED=1 is set. The compound
   deploy targets (deploy-all, deploy-allX) explicitly call validate
   ONCE at the top, then pass VALIDATED=1 to every $(MAKE) sub-target
   invocation. The 9 deploy-* targets that have `validate` as a prereq
   still validate when invoked directly (current behavior preserved),
   but no longer re-validate inside a compound run.

   Entire recipe body is now one shell invocation (with line
   continuations) so the early-exit actually terminates the recipe —
   each @-line is otherwise its own sub-shell and `exit 0` in one
   doesn't stop the next.

2) Output compression. Per-template "Validating X..." echoes (15+
   lines) replaced by a single summary line:
     "Validating CFN templates + param files... ✓ 11 templates + 7 param files"
   cfn-lint / jq output is captured into a temp file and only
   surfaced on failure (with a per-file ✗ marker).

Result:
  - deploy-all:  ~80 lines of validation noise → 1 line
  - deploy-allX: ~150 lines → 1 line

Tradeoff: editing a template *during* a long deploy-all run won't
re-trigger validation on subsequent sub-targets. In practice we don't
edit templates mid-deploy; if you want the old behavior for a one-off
run, just don't use the compound targets.

Co-Authored-By: Claude Opus 4.7 (1M context) <noreply@anthropic.com>
```
