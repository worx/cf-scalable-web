## About `restore-deploy-host` and `list-deploy-host-backups`

I deliberately did NOT add preflight to those two:

| Target | Preflight added? | Reason |
|--------|-------------------|--------|
| `deploy-deploy-host` | ✅ Yes | Needs bucket for UserData auto-restore + Scheduler to work |
| `backup-deploy-host` | ✅ **Yes (just added)** | Needs bucket to write to |
| `restore-deploy-host` | ❌ No | If bucket doesn't exist, there's nothing to restore — auto-creating an empty bucket doesn't help |
| `list-deploy-host-backups` | ❌ No | Informational query; "bucket doesn't exist" is a valid answer |

If you disagree and want ALL of them to auto-deploy the bucket, easy to add. But asymmetric feels right to me — auto-deploy where the missing-bucket case is a real blocker, error-out where the missing-bucket case is genuinely informative.
