## Architecture

Two stacks by design:
  cf-deploy-host          (compute - ephemeral by nature)
  cf-deploy-host-backups  (S3 bucket - long-lived by design)

Kept separate because the backups bucket's whole purpose is to survive
events that destroy the deploy-host compute. Merging them would defeat
the safety net.

`make deploy-deploy-host` auto-deploys the backups bucket if it's not
present (preflight check), so operators never have to remember the
two-step bootstrap. `make destroy-deploy-host-backups` is intentionally
never wrapped anywhere - manual-only, requires typing
"DESTROY ALL DEPLOY-HOST BACKUPS" verbatim.
