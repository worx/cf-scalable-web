Good news: the new instance is functionally identical. The SG ID stayed the same (it's a different resource), so the new export value should now be the real `sg-xxxxxxxxx`.

Verify before continuing:
```bash
aws cloudformation list-exports \
    --query 'Exports[?Name==`cf-deploy-host-sg-id`].Value' \
    --output text
```

That should show `sg-xxxxxxxxx` not `cf-deploy-host-sg`. If yes, you're good for `make deploy-peering ENV=sandbox`.

Worth a memory note: **`make deploy-deploy-host` always replaces the instance**, so don't run it from the deploy-host shell unless you're ready to lose the session. That's a gotcha worth saving.
