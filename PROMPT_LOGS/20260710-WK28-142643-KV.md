### Safety features baked in

- **Credentials never in `ps` output** — uses `--defaults-extra-file` with a mode-600 temp file
- **Credfile shredded on exit** (via `shred -u` if available, `rm -f` fallback) — chained with `log_upload_and_exit` in a single trap
- **Placeholder guard** — refuses to run if secret's password field still starts with `PLACEHOLDER-`
- **Pipefail-aware** — if mysqldump errors mid-stream, we detect it and remove the partial S3 object
- **Source metadata captured** — table count and size printed for cross-check against restore-mysql's verification later
