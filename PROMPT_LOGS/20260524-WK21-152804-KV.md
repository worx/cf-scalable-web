### 15:29:54 - Response

🎉 **30:58 deploy / 17:23 destroy — full cycle in under 50 min.** From the original ~60-80 min sequential, that's a solid 30-40% improvement. Tight numbers.

**On the naming consolidation: I'd vote yes.** Three variants is the right number, and the current naming (`deploy-all`, `deploy-allX`, `deploy-allXX`) is opaque to anyone who didn't watch them get built. `TYPE=` flags map directly to operator intent.

**My take on the names:**

- `TYPE=parallel` (default) — clear, accurate, no ambiguity
- `TYPE=linear` — also clear. "Sequential" works too but is longer. Keep `linear`.
- `TYPE=minimal` — works, but "minimal" is a relative word. The thing that makes that variant *different* is it stops before the data layer (no RDS, no cache, no app-drupal). Two alternatives worth considering:
  - **`TYPE=no-data`** — most precise. Says exactly what's missing.
  - **`TYPE=compute`** — what you actually get. The variant deploys foundation + image-builder + AMIs + compute. "I want compute infra running, nothing else."
  
  I'd lean toward **`TYPE=no-data`** because the name immediately tells you why you'd pick it ("I want infra without the data layer for some compute-only test"). But `minimal` is fine and shorter. Your call.

**Proposed structure:**

| Public | Dispatches to | Old equivalent |
|--------|---------------|----------------|
| `make deploy-all` (default) | `deploy-all-parallel` | `deploy-allXX` |
| `make deploy-all TYPE=parallel` | `deploy-all-parallel` | `deploy-allXX` |
| `make deploy-all TYPE=linear` | `deploy-all-linear` | `deploy-allX` |
| `make deploy-all TYPE=no-data` (or `minimal`) | `deploy-all-no-data` | `deploy-all` (original) |
| `make destroy-all` (default) | `destroy-all-parallel` | `destroy-allXX` |
| `make destroy-all TYPE=linear` | `destroy-all-linear` | `destroy-all` (original sequential) |

**Backward compatibility:**
- `deploy-allXX`, `deploy-allX`, `destroy-allXX` keep working as **aliases** to the new canonical names (for muscle memory and for any script Kurt has elsewhere)
- The old `deploy-all` target gets renamed internally — but if you type `make deploy-all` with no TYPE, you get the new parallel default (which is what most operators actually want)

**Help text consolidation:**

```
make deploy-all                    Parallel full-stack deploy (default; ~30 min)
make deploy-all TYPE=linear        Sequential full-stack (~45-50 min; old behavior)
make deploy-all TYPE=no-data       Foundation + compute only — no RDS/cache/app-drupal
make destroy-all                   Parallel destroy (default; ~17 min)
make destroy-all TYPE=linear       Sequential destroy (older behavior, ~25 min)
```

Help text drops from 5 lines mentioning cryptic `X`/`XX` suffixes to 5 lines that tell you what each one actually does.
