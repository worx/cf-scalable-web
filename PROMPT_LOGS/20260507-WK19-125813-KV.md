🎉 SUCCESS marker! Big win.

Here's the SSH-over-SSM setup:

## On your Mac — `~/.ssh/config`

```
# SSM-over-SSH proxy for any AWS instance ID
Host i-* mi-*
  User ubuntu
  IdentityFile ~/.ssh/id_ed25519
  StrictHostKeyChecking accept-new
  ProxyCommand sh -c "AWS_PROFILE=ZI-Sandbox aws ssm start-session --target %h --document-name AWS-StartSSHSession --parameters 'portNumber=%p'"
```

Replace `~/.ssh/id_ed25519` with whichever key you actually use, and `AWS_PROFILE=ZI-Sandbox` if your default profile is correct (or remove that bit and rely on the shell's `AWS_PROFILE`).

## Prerequisite — your public key must be on the deploy-host

The deploy-host has no inbound port 22 rule, but more importantly, no `authorized_keys` for the `ubuntu` user. SSH-over-SSM still needs the SSH layer to authenticate. SSM in once and add your key:

```bash
# In an SSM session on the deploy-host:
mkdir -p ~/.ssh && chmod 700 ~/.ssh
echo "ssh-ed25519 AAAAC3... user@your-mac" >> ~/.ssh/authorized_keys
chmod 600 ~/.ssh/authorized_keys
```

Get your public key from `~/.ssh/id_ed25519.pub` (or your equivalent) on the Mac.

## Prerequisite — `session-manager-plugin` on your Mac

```bash
brew install session-manager-plugin    # if not already installed
```

## Then SSH and SCP just work

```bash
# Find current instance ID
aws ec2 describe-instances \
  --filters "Name=tag:Name,Values=cf-deploy-host" "Name=instance-state-name,Values=running" \
  --query 'Reservations[].Instances[].InstanceId' --output text

# SSH (replace with actual ID)
ssh i-0abc123def456

# SCP the bootstrap log to your Mac
scp i-0abc123def456:/var/log/deploy-host-bootstrap.log /tmp/bootstrap.log
```

Then `cat /tmp/bootstrap.log | head -200` (or paste the file contents into our conversation, or just summarize what's interesting in it).

## Alternative if the SSH dance is annoying

Since your deploy-host has AdministratorAccess via instance role, you can just dump the log to S3 and grab it from there:

```bash
# On deploy-host:
aws s3 cp /var/log/deploy-host-bootstrap.log s3://YOUR-BUCKET/logs/deploy-host-bootstrap.log

# On Mac:
aws s3 cp s3://YOUR-BUCKET/logs/deploy-host-bootstrap.log /tmp/
```

Faster for one-off transfers. The SSH setup is worth doing eventually for ongoing work though.
