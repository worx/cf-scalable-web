### 17:01:22 - Response

Yes — exactly. The deploy-host is the right place for these commands. Specifically:

- **`make build-amis`** is the big one: Image Builder pipelines take ~30-60 min and you don't want to tie up your Mac terminal. Run it on the deploy-host with `nohup ... &` or just leave the SSM session open.
- **`sync-configs`, `deploy-image-builder`, `deploy-ami-params`, `deploy-compute`** all just need AWS credentials and the repo. The deploy-host has both via its instance role and the auto-cloned repo. Running there is more consistent.

**Standard caveat from the gotchas doc:** never run `make deploy-deploy-host` *from* the deploy-host (your SSM session dies with the box) — but none of these commands touch the deploy-host stack, so you're fine.

**Don't forget to pull first** — the deploy-host's clone is on commit `1e48a92` (or thereabouts), it needs `86a1fe5`:

```bash
cd ~/cf-scalable-drupal && git pull
make sync-configs ENV=sandbox
make deploy-image-builder ENV=sandbox
make build-amis ENV=sandbox          # ~30-60 min, can run in background
make deploy-ami-params ENV=sandbox
make deploy-compute ENV=sandbox
```
