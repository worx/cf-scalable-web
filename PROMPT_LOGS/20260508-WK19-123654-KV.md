## What to watch for

The script's `step` markers should land in this order:

1. ✓ Drupal Cloud Install Starting
2. ✓ Fetch credentials (RDS master + create/read Drupal secrets)
3. ✓ Create drupal_user in PostgreSQL
4. ✓ Create directories on FSx
5. ✓ composer create-project (~2 min — slowest step)
6. ✓ composer require drush/drush
7. ✓ Generate (or reuse) hash_salt
8. ✓ drush site:install (~1 min)
9. ✓ Replace settings.php with env-var-driven config
10. ✓ Verify install via drush
11. ✓ Drop install marker

If it fails on a specific step, paste the surrounding output and we'll diagnose. The most likely friction points:

| Step | Possible issue | Fix |
|------|---------------|-----|
| 2 (credentials) | `worxco/sandbox/rds/master-password` doesn't exist | Was it created during `deploy-database`? Check: `aws secretsmanager list-secrets --query 'SecretList[?starts_with(Name, \`worxco/sandbox\`)].Name'` |
| 3 (db user) | psql connection fails | Peering not working? `nc -zv $RDS_ENDPOINT 5432` |
| 4 (FSx dirs) | Permission denied on `mkdir -p` | FSx mounted but `/var/www/sandbox` is root-owned and your user can't sudo? |
| 5 (composer) | OOM or network | t4g.small has 2GB RAM; composer typically OK |
| 8 (drush) | Database connection error | Wrong db user perms, or db doesn't exist |
