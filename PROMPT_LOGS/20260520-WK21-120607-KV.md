fix(bootstrap): MOTD print belongs in /etc/profile.d + /etc/zsh/zshrc.d, not ~/.bashrc

Previous code appended the `cat /etc/motd; export MOTD_SHOWN=1` snippet
to /home/ubuntu/.bashrc. That file is read by interactive non-login bash
shells. SSM Session Manager invokes `bash -l` (a login shell), which
does NOT read ~/.bashrc — it reads /etc/profile, which sources
/etc/bash.bashrc, which then auto-execs into `zsh -l`. zsh reads
/etc/zsh/zshrc.d/*.zsh + ~/.zshrc. None of those paths touch ~/.bashrc.
Net effect: the MOTD print was dead code; operators never saw the
project's "you're on the deploy-host, here are the things you might
want to do" message.

Fix: drop the print into two system-wide files instead:

  - /etc/profile.d/00-worxco-motd.sh — read by login bash via
    /etc/profile (covers the NO_AUTO_ZSH=1 escape-hatch case).
  - /etc/zsh/zshrc.d/00-motd.zsh — read by interactive zsh (the
    normal SSM flow after the bash→zsh auto-exec).

Both guarded by `MOTD_SHOWN` env var (which carries across the exec
from bash to zsh) so the rare both-fire case doesn't double-print.
Both guarded by `-t 1` so non-interactive shell invocations don't
spam MOTD into command output.
