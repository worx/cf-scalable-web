Both diagnostics look right — pgsql extension is loaded and env vars are populated. So it's not the obvious things. Time to look deeper.

## Most likely cause — the pgsql driver files were deleted by rsync --delete

Drupal 11 doesn't bundle both database drivers in core the way older versions did. When we did:

```bash
sudo rsync -av --delete /var/www/drupal-staging/ /var/www/drupal/
```

`--delete` removed any files that existed in the destination but NOT in the source. Since React2's prod codebase uses **MySQL**, its file tree doesn't include the PostgreSQL driver files. If sandbox had them (from `install-drupal-full`), they got wiped out.

That would explain everything:
- Codebase loads fine (drush shows version, root, PHP)
- Bootstrap fails silently when Drupal tries to load the pgsql driver classes
- No error message from drush status because drush just skips DB fields it can't populate

## Verify with 3 quick checks

```bash
# 1. Verbose drush — tells us the actual error at bootstrap
./vendor/bin/drush -vv status 2>&1 | tee /tmp/drush-debug.log
head -100 /tmp/drush-debug.log

# 2. Look for the pgsql driver module in Drupal 11's expected locations

# 3. Check what's in modules/contrib (Drupal 11 moves pgsql driver here)
ls /var/www/drupal/web/modules/contrib/ 2>/dev/null
```

**What we're hoping to see (healthy state):**

- `find` should show at least `/var/www/drupal/web/modules/contrib/pgsql` (the community-maintained PostgreSQL driver module) or `/var/www/drupal/web/core/modules/pgsql` if it's still in core in your specific version
- `ls modules/contrib` should include `pgsql` in the list

**What we probably see (broken state):**

- `find` returns nothing or only unrelated matches
- `ls modules/contrib` has your client's contrib modules but NO `pgsql` directory
- verbose drush shows an error like `RuntimeException: Missing database driver namespace` or `Class 'Drupal\pgsql\Driver\Database\pgsql\Connection' not found`

## If that's the situation — two clean recovery paths

**Recovery A: Composer install the pgsql driver module (proper way)**

```bash
cd /var/www/drupal
sudo -u ubuntu composer require drupal/pgsql
```

This adds it to `composer.json`, downloads to `web/modules/contrib/pgsql/`, and registers autoloading. Requires composer installed on deploy-host. Then:

```bash
sudo chown -R www-data:www-data /var/www/drupal
sudo find /var/www/drupal -type d -exec chmod 755 {} \;
sudo find /var/www/drupal \
  -path '*/web/sites/default/files' -prune -o \
  -type f -exec sh -c 'head -c2 "$1" | grep -q "^#!" && chmod +x "$1"' _ {} \; 2>/dev/null

./vendor/bin/drush status   # try again
```
