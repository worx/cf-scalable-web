### 11:44:17 - Response

The official source is the **AWS CloudFormation User Guide** — every resource type has a documentation page with a "Return values" section that lists both the `Ref` return and all `GetAtt` attributes.

For FSx specifically:
`https://docs.aws.amazon.com/AWSCloudFormation/latest/UserGuide/aws-resource-fsx-filesystem.html`

Scroll to the **Return values** section and you'll see:
- **Ref**: Returns the file system ID
- **Fn::GetAtt**: Lists `DNSName`, `RootVolumeId`, `LustreMountName` (for Lustre type), etc.

The pattern for any resource type is the same URL structure:
`https://docs.aws.amazon.com/AWSCloudFormation/latest/UserGuide/aws-resource-{service}-{resource}.html`

For example:
- EC2 instance: `aws-resource-ec2-instance.html` — attributes include `PrivateIp`, `PublicIp`, `AvailabilityZone`
- RDS: `aws-resource-rds-dbinstance.html` — attributes include `Endpoint.Address`, `Endpoint.Port`
- ElastiCache: `aws-resource-elasticache-replicationgroup.html` — attributes include `PrimaryEndPoint.Address`, `PrimaryEndPoint.Port`

The fastest way to get there is just to Google: **"cloudformation aws::fsx::filesystem return values"** — the AWS docs page is always the top result.

There's no single master list of all attributes across all resource types — each resource page documents its own. But the "Return values" section is always in the same spot on every page.
