### What `cf-deploy-peering.yaml` does

1. Creates VPC peering connection: default VPC ↔ project VPC
2. Adds route in default VPC main route table: `<project-cidr>` → peering
3. Adds routes in project VPC route tables (public + 1-3 private depending on `AZCount`): `172.31.0.0/16` → peering
4. Adds 4 SG ingress rules:
   - FSx SG ← deploy-host SG on TCP 2049 (NFS)
   - FSx SG ← deploy-host SG on TCP 111 (portmapper)
   - RDS SG ← deploy-host SG on TCP 5432
   - Valkey SG ← deploy-host SG on TCP 6379

### How `make deploy-peering` works

The Make target:
1. Looks up the default VPC ID via `aws ec2 describe-vpcs --filters Name=isDefault,Values=true`
2. Looks up the main route table ID for that VPC
3. Passes both as `--parameter-overrides` to CloudFormation (no Lambda, no hand-edits)

### How `make test-peering` works

1. Finds the deploy-host EC2 instance (must be running)
2. Pulls FSx/RDS/Valkey endpoints from existing stack outputs
3. Runs `nc -zv` on each via SSM send-command
4. Reports the output
