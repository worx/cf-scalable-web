docs(architecture): correct VPC diagram + remove stale NAT Gateway refs

The VPC Layout diagram and its "Key Points" footnote claimed NAT Gateways
provide outbound internet for OS updates / package installs. That's
wrong on two counts:

1. No environment actually deploys NAT Gateways. cf-vpc.yaml has them

   staging explicitly set it false, and production only sets the HA
   flag without the base flag (no-op). Live AWS confirms zero NAT
   Gateways exist anywhere in the account.

2. The whole point of this architecture is privatized compute. No
   outbound internet from running instances. OS updates and package
   installs happen at AMI build time (Image Builder), not on running
   boxes. Drupal codebase installs happen on the deploy-host (separate
   VPC, full internet) — never on the compute fleet. Outbound AWS API
   calls use VPC Endpoints, which AWS-internally route traffic without
   the public internet.

    added VPC Endpoints subgraph (SSM/SSM Messages/EC2 Messages,

    AWS API via VPC Endpoints. Called out that drupal.org update
