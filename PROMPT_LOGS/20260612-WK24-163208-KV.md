## How to run it (sketch вҖ” assume pgloader on the sandbox deploy-host)

**Window 1: Open the SSM port-forwarding tunnel**

From the deploy-host shell (which needs the `ZoningInfoAdmin` profile available вҖ” see "Credentials question" below):

```bash
# Get the jumpbox instance ID from your Mac and SSM it in, or fetch from CFN here.
JUMPBOX_ID=$(aws --profile ZoningInfoAdmin cloudformation describe-stacks \
  --stack-name cf-migration-jumpbox \
  --query "Stacks[0].Outputs[?OutputKey=='JumpboxInstanceId'].OutputValue" \
  --output text)

aws --profile ZoningInfoAdmin ssm start-session \
  --target "$JUMPBOX_ID" \
  --document-name AWS-StartPortForwardingSessionToRemoteHost \
  --parameters '{"host":["d8demo-cluster.cluster-cmbywichztfj.us-east-1.rds.amazonaws.com"],"portNumber":["3306"],"localPortNumber":["3306"]}'
```

This blocks (it's a foreground tunnel). Leave it running.

**Window 2: Run pgloader against the local tunnel + sandbox RDS**

```bash
source /etc/worxco/envs/sandbox      # populates DRUPAL_DB_HOST, _USER, _PASS, _NAME

# Save a pgloader load file with Drupal-aware casts (the casts handle the
# MySQLвҶ’Postgres type translations Drupal needs: TINYINT(1)вҶ’BOOLEAN,
# zero-datesвҶ’NULL, DATETIMEвҶ’TIMESTAMPTZ, etc.)
cat > /tmp/zinew.load <<EOF
LOAD DATABASE
  FROM      mysql://worxco:PROD_PASSWORD@localhost:3306/zinew
  INTO      postgresql://$DRUPAL_DB_USER:$DRUPAL_DB_PASS@$DRUPAL_DB_HOST:5432/$DRUPAL_DB_NAME

WITH include drop, create tables, create indexes, reset sequences, foreign keys,
     downcase identifiers

CAST type tinyint when (= 1 precision) to boolean drop typemod using tinyint-to-boolean,
     type datetime to timestamptz drop default drop not null using zero-dates-to-null,
     type date drop not null drop default using zero-dates-to-null,
     type timestamp drop default drop not null using zero-dates-to-null
;
EOF

pgloader /tmp/zinew.load
```

This is the standard "Drupal MySQL вҶ’ PostgreSQL via pgloader" recipe. Iteration on the first run will probably want to tweak the casts based on what your specific Drupal version has.
