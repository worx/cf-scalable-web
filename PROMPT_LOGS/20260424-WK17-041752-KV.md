The region is empty — `https://ssm..amazonaws.com` (double dot, no region). The configure-php script is calling `aws ssm get-parameter --region "$REGION"` but `$REGION` is empty. The script gets the region from the instance metadata, but the metadata call is probably failing because there's no outbound internet from the private subnet.

Also confirms: `SSM Agent unable to acquire credentials` — no outbound connectivity from the private VPC subnets (no NAT gateway, no VPC endpoints).

This is a fundamental networking issue — the PHP and NGINX instances in private subnets can't reach AWS APIs. They need either:
1. **NAT Gateway** (expensive, ~$32/mo per AZ)
2. **VPC Endpoints** for the services they need (SSM, Secrets Manager, S3, CloudWatch)

This was always part of the architecture but was never deployed. For now, the quickest path to get the boot scripts working is to add VPC endpoints for the essential services. Want me to check what's in the VPC template for this, or should we pick this up fresh tomorrow?
