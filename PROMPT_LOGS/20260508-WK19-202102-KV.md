### 20:21:19 - Response

Two bad pieces in my command — sorry:
1. `imagePipeline.status` is just ENABLED/DISABLED (the pipeline's *own* state), not build state. Need `list-image-pipeline-images` to see the in-flight build.
2. `describe-images` doesn't support a `creationDate` filter. Have to sort and slice client-side.

Corrected:

```bash
# Three latest pipeline executions, one line per pipeline
for PIPE in $(aws imagebuilder list-image-pipelines \
    --query 'imagePipelineList[?contains(name,`sandbox`)].arn' --output text); do
  NAME=$(echo "$PIPE" | awk -F'/' '{print $NF}')
  printf "%-30s " "$NAME"
  aws imagebuilder list-image-pipeline-images --image-pipeline-arn "$PIPE" \
    --query 'reverse(sort_by(imageSummaryList, &dateCreated))[0].[state.status,dateCreated]' \
    --output text
done

# Recent AMIs (top 6) — sorted by creation, no date filter
aws ec2 describe-images --owners self \
  --filters "Name=tag:Environment,Values=sandbox" \
  --query 'reverse(sort_by(Images, &CreationDate))[:6].[Name,State,CreationDate,ImageId]' \
  --output table
```

When you see all three pipelines flip to `AVAILABLE` (and three new AMIs with `2026-05-09T01-08-*` timestamps appear at the top of the AMI list), then run:

```bash
make deploy-ami-params ENV=sandbox
make deploy-compute ENV=sandbox
```

If you want to leave it polling, this one-liner exits when all three reach a terminal state (AVAILABLE or FAILED):

```bash
until [ "$(for P in $(aws imagebuilder list-image-pipelines \

  aws imagebuilder list-image-pipeline-images --image-pipeline-arn "$P" \
    --query 'reverse(sort_by(imageSummaryList, &dateCreated))[0].state.status' --output text; \
done | sort -u | tr '\n' ',')" = "AVAILABLE," ]; do
  date '+%H:%M:%S'; \

done; echo "ALL AVAILABLE — ready to deploy-ami-params + deploy-compute"
```
