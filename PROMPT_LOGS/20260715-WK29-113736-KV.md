## Design highlights

- **Manifest file** inside every backup archive (`BACKUP-MANIFEST.txt`) documenting hostname, instance ID, timestamp, paths included/excluded — makes forensics easier later
- **Stub file logic** in restore: writes clear `.aws/credentials` placeholder if missing after restore, so operator gets an obvious "get creds from Mac" reminder instead of silent "profile not found"
- **Sourcing pattern** for `_common.sh`: scripts look next-to-self first, fall back to `migration/scripts/_common.sh` — works whether shipped standalone or from repo
- **EventBridge Scheduler** with `America/Chicago` timezone matches jumpbox pattern (DST-aware)
- **Pre-destroy backup** in `destroy-deploy-host` with graceful degradation if backups bucket isn't deployed yet
- **Adminstrator role already covers all needed permissions** — no IAM policy additions needed on deploy-host role (it already has AdministratorAccess for the sandbox account)

## Critical operational order — READ BEFORE DEPLOYING

**The updated `cf-deploy-host.yaml` includes UserData changes.** Same "Conditional replacement" scenario we hit with the jumpbox. Do these in strict order:

```bash
# 1. Deploy the backups bucket FIRST (no risk — brand new stack, no existing resources)
make deploy-deploy-host-backups

# 2. Take a manual backup of your CURRENT deploy-host state
#    (uploads scripts to S3, creates first archive)
make backup-deploy-host

# 3. Verify backup landed
make list-deploy-host-backups
# should show config/deploy-host-latest.tar.gz + one timestamped archive

# 4. Preview the cf-deploy-host CFN update BEFORE running it
#    (learned this lesson the hard way with the jumpbox)
aws cloudformation deploy \
    --stack-name cf-deploy-host \
    --template-file cloudformation/cf-deploy-host.yaml \
    --capabilities CAPABILITY_NAMED_IAM \
    --no-execute-changeset

# then describe the change set:
aws cloudformation describe-change-set \
    --change-set-name <changeSet-arn-from-above> \
    --query 'Changes[*].ResourceChange.{Action:Action,LogicalId:LogicalResourceId,Type:ResourceType,Replacement:Replacement}' \
    --output table

# 5. If change set shows Jumpbox=Conditional-or-True on the EC2 instance, EXPECT REPLACEMENT.
#    Your backup from step 2 will auto-restore. Proceed:
make deploy-deploy-host

# 6. Verify auto-restore worked
make ssm-deploy-host
# once inside:
ls -la /root/.ssh/
cat /home/ubuntu/.aws/config    # should have your profile definitions
cat /home/ubuntu/.aws/credentials  # should show the STUB (real creds still on Mac)
```
