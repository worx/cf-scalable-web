
The new file format starts each line with `export ` so `^FSX_DNS=` doesn't match anymore. With `set -e` + `set -o pipefail`, an unmatched grep can kill the script under some bash versions. If that's the cause, the bootstrap would have died right before the MOTD step.

The fix (after we confirm via the log) is straightforward — the grep needs to handle the `export` prefix:

```bash
# Old:  grep '^FSX_DNS=' "$envfile"   (matches 'FSX_DNS=...')
# New:  grep '^\(export \)\?FSX_DNS=' "$envfile" | sed 's/^export //'   (matches both)
# Even better: just source the file in a subshell
```
