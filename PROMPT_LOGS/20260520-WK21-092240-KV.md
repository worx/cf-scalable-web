## On Drush aliases / Aegir-style site config — what I found

You're remembering Aegir's `.drushrc.php` per-site pattern. Here's how it maps to modern stack:

- **Modern Drush (10+)** uses YAML alias files: `<sitename>.site.yml` in `drush/sites/`, `/etc/drush/sites/`, or `~/.drush/sites/`. Same idea, different syntax.
- **What goes in an alias file**: `root` (path to Drupal install), `uri` (site URL), `options` (db-url, etc.). Drush auto-discovers any site listed in these directories.
