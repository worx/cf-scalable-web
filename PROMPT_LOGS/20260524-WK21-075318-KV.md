fix(https): admin-login-url + endpoints use https://<site-name>; tmux OSC 52

Three related operator-experience fixes.

1. admin-login-url.sh: drush user:login was called with
     --uri=http://$ALB_DNS
   which produced a URL using the ALB's amazonaws.com name AND the
   HTTP scheme. Two problems: (a) the ACM cert doesn't cover the ALB's
   amazonaws.com name, so manually changing http→https in the URL
   triggers a cert mismatch in the browser; (b) HTTP-only URLs are
   wrong for any env that has HTTPS configured.

   Fix: detect HTTPS by checking for the AlbCertificateArn output on
   the compute-alb stack. If present, build --uri=https://<site-name>
   using the SITE_NAME from SSM. If absent (HTTP-only env), fall back
   to --uri=http://<alb-dns>. Operator gets a working URL either way.

2. Makefile `endpoints` target: same HTTP-vs-HTTPS detection. When the
   env has HTTPS enabled, all printed URLs (Browser URL, Login URL,
   curl test) use https://<site-name>. When HTTPS is off, falls back
   to the previous http://<alb-dns> output. Removed the now-stale
   "Browser URL: http://ALB_DNS — note our trusted_host_patterns
   allows that..." preamble; with HTTPS on, that's not the right
   guidance anymore.

3. deploy-host /etc/tmux.conf (via bootstrap.sh): adds
     set -g set-clipboard on
   which makes tmux push its copy buffer through OSC 52 escape
   sequences to the host's clipboard. Click-drag selections inside
   tmux (which `mouse on` captures, so the highlight previously
   vanished on mouse-release) now end up in the operator's system
   clipboard. Plus a `MouseDragEnd1Pane` binding so the
