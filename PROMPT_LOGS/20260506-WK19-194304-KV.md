## When to Apply

Since `make deploy-deploy-host` replaces the EC2 instance, **don't** run this from the deploy-host shell while `make deploy-all` is in progress. You'd kill the very session running deploy-all.

Sequence:
1. Wait for `make deploy-all` to finish (~60-90 min)
2. Run `make deploy-deploy-host` from your Mac (not from the deploy host)
3. SSM into the new deploy-host
4. Verify: `which composer drush psql redis-cli`, `ls /etc/worxco/`, `mount-env --help` (well, `mount-env` with no args shows usage)
