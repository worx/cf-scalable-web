2. **Refuses to proceed if `site-name` ends in `.test`** — explains that's a dummy hostname that won't resolve publicly, and tells you to update the param file and redeploy app-drupal

This is the most likely failure mode and is easy to fix once you know the signature.

Help text in `make help` updated to differentiate the two:
```

make smoke-test-public ENV=...     End-to-end via the real public DNS name
```

The two together form a useful pair: `smoke-test-drupal` validates the Drupal install itself (works even before DNS is set up), `smoke-test-public` validates the full public-URL path.
