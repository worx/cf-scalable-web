### 21:26:24 - Response

That's the bash continuation lines getting mangled by tmux paste. The `command substitution: line 121` error and then just timestamps means the inner `$(...)` parse fails, returns empty, the test compares `""` to `"AVAILABLE,"` (always false), and the loop just runs the body (date + sleep).

Let me give you something simpler that's robust to paste issues. **Kill the broken loop with Ctrl-C**, then run this:

```bash
cat > /tmp/wait-amis.sh <<'EOF'
#!/bin/bash
while true; do
  STATUSES=""
  for P in $(aws imagebuilder list-image-pipelines \
      --query "imagePipelineList[?contains(name,'php-fpm') && contains(name,'sandbox')].arn" \
      --output text); do
    NAME=$(echo "$P" | awk -F/ '{print $NF}')
    S=$(aws imagebuilder list-image-pipeline-images --image-pipeline-arn "$P" \
        --query "reverse(sort_by(imageSummaryList, &dateCreated))[0].state.status" \
        --output text)
    printf "  %-30s %s\n" "$NAME" "$S"
    STATUSES="${STATUSES}${S},"
  done
  date '+now=%H:%M:%S'
  if [ "$STATUSES" = "AVAILABLE,AVAILABLE," ]; then
    echo "READY"
    break
  fi
  echo "---"
  sleep 30
done
EOF
chmod +x /tmp/wait-amis.sh
/tmp/wait-amis.sh
```

It writes the script to disk first (no continuation/paste hazards), then runs it. Each tick prints both pipeline names with their status, plus the time, so you can see actual progress instead of just timestamps. Exits with `READY` when both PHP pipelines are AVAILABLE.
