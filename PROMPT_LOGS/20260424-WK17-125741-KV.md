The PHP role only has S3 access to `sandbox-drupal-media` bucket. The `sandbox-image-builder-kv-worxco` bucket isn't in the policy. 

Quickest fix — just use the media bucket instead. From the deploy host, upload to the media bucket:

```bash
aws s3 cp /tmp/composer-setup.php s3://sandbox-drupal-media-kv-worxco/tools/composer-setup.php
```

Then on the PHP instance:

```bash
aws s3 cp s3://sandbox-drupal-media-kv-worxco/tools/composer-setup.php /tmp/composer-setup.php
sudo php /tmp/composer-setup.php --install-dir=/usr/local/bin --filename=composer
```

Or — we could add the image-builder bucket to the PHP role, but that's a broader change. The media bucket workaround gets you Composer right now.
