Update install-drupal.sh, remove-drupal.sh, and cf-app-drupal.yaml to

Drupal lives at /var/www/drupal on every host (deploy-host with use-env

parameter now defaults to /var/www/drupal regardless of env.

  - INSTALL_DIR=/var/www/drupal (was /var/www/$ENV/drupal)
  - Verify use-env was run for the matching env before proceeding
    (single-mount model means a sandbox install could silently land on
    staging's FSx if the operator forgot to switch)
  - chown to ubuntu:www-data + chmod g+rX so the runtime fleet's
    PHP-FPM workers (running as www-data) can read sensitive files
    like salt.txt without further per-file fiddling

remove-drupal.sh:
  - Same path updates
  - Refuse to run if active env doesn't match the env passed in

cf-app-drupal.yaml:

  - DrupalInstallPath output: '/var/www/drupal'

Deployment notes:
  - `make deploy-app-drupal ENV=sandbox` updates the SSM parameter
  - install-drupal.sh reads SSM at runtime, so existing AMIs pick up
    the new path on next install with no AMI rebuild needed
  - The existing FSx contents (currently at /fsx/drupal/) don't move
    — only the mount target on the deploy-host changes, from
    /var/www/sandbox to /var/www. The runtime fleet was already on
