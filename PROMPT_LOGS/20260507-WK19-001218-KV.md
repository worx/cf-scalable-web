fix(deploy-host): Set HOME=/root in UserData so composer can run

UserData runs via cloud-init with HOME unset. Composer requires HOME
(or COMPOSER_HOME) to operate — even the official installer needs it
for cache directories. Symptom in /var/log/deploy-host-bootstrap.log:

  All settings correct for using Composer
  The HOME or COMPOSER_HOME environment variable must be set for composer
  to run correctly

With set -euo pipefail at the top of UserData, that error aborts the
entire script. Result: composer (and everything after it — drush,
session-manager-plugin, /etc/worxco, helper scripts, MOTD) silently
never runs. The instance comes up looking healthy but missing every
Phase B addition.

Fix: export HOME=/root near the top of UserData (alongside
DEBIAN_FRONTEND=noninteractive), so it applies to all subsequent
steps including composer.

Also: replaced the brittle 'composer --version' verify echo with a
plain 'ls -la /usr/local/bin/composer'. The verify added no real
value (the install command's exit code already tells us if it worked)
and was a second place HOME caused failures.

For deploy hosts deployed BEFORE this commit, you can either:

  1. destroy/redeploy the deploy-host (cleanest):
       make destroy-peering ENV=sandbox
       make destroy-deploy-host
       make deploy-deploy-host
       make deploy-peering ENV=sandbox

  2. fix in place by running the post-composer steps manually:
       cd ~/projects/cf-scalable-web && git pull
       export HOME=/root
       sudo -E curl -sS https://getcomposer.org/installer -o /tmp/composer-setup.php
       sudo -E php /tmp/composer-setup.php --install-dir=/usr/local/bin --filename=composer
       sudo rm /tmp/composer-setup.php
       composer global require drush/drush
       echo 'export PATH="$HOME/.config/composer/vendor/bin:$PATH"' >> ~/.bashrc
       sudo mkdir -p /etc/worxco/envs
       echo deploy-host | sudo tee /etc/worxco/deploy-host-marker
       cd ~/projects/cf-scalable-web
       sudo install -m 0755 scripts/deploy-host/info-env           /usr/local/bin/info-env
       sudo install -m 0755 scripts/deploy-host/show-env           /usr/local/bin/show-env
       sudo install -m 0755 scripts/deploy-host/psql-env           /usr/local/bin/psql-env
       sudo install -m 0755 scripts/deploy-host/valkey-env         /usr/local/bin/valkey-env
       sudo install -m 0755 scripts/deploy-host/mount-env          /usr/local/sbin/mount-env
       sudo install -m 0755 scripts/deploy-host/refresh-env-config /usr/local/sbin/refresh-env-config
       sudo install -m 0440 scripts/deploy-host/worxco-refresh-env-config.sudoers /etc/sudoers.d/worxco-refresh-env-config
       sudo refresh-env-config
