### Phase C — Drupal Stack + Make Targets

**`cf-app-drupal.yaml`** — minimal stack, mostly stores configuration:

```
Parameters:
  - EnvironmentName
  - DrupalVersion (default: 11)
  - DrupalAdminEmail
  - DrupalDatabaseName (default: drupal_${EnvironmentName})

Resources:
  - DrupalAdminSecret (Secrets Manager) — admin password generated at create time
  - DrupalDbSecret (Secrets Manager) — Drupal DB user password (not RDS master)
  - SSM Parameters:
    - /${env}/drupal/db-name
    - /${env}/drupal/db-user
    - /${env}/drupal/site-url

Outputs:
  - SiteURL (the ALB DNS)
  - InstallCommand (e.g., "make install-drupal ENV=sandbox")
  - EtcHostsLine (a ready-to-paste suggestion if you want a friendly hostname later)
```

The stack itself doesn't run the install. It just stores what we need. The install is triggered by `make install-drupal`, which:

1. SSM-execs onto deploy-host
2. Verifies FSx is mounted at `/var/www`
3. Checks for `/var/www/drupal/.installed` marker → errors if present
4. Creates Drupal DB user + database in RDS using master credentials
5. `composer create-project drupal/recommended-project /var/www/drupal --no-interaction`
6. `drush site:install standard` with DB connection from SSM/Secrets
7. Writes `/var/www/drupal/web/sites/default/settings.php` reading endpoints from env vars
8. Drops marker file
9. Reports the ALB DNS URL to visit
