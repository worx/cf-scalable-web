### 13:01:51 - Response

Yes — SSH over SSM works. The SSM agent is running and SSH is listening on the instance (sshd starts by default on Ubuntu). There's no inbound port needed because the tunnel goes through SSM.

You can use it for SCP too:

```bash
scp -o ProxyCommand="aws ssm start-session --target %h --document-name AWS-StartSSHSession --parameters portNumber=%p" composer-setup.php ubuntu@i-INSTANCEID:/tmp/
```

But you'd need an SSH key on the instance for that. The PHP instances don't have your SSH key.

The simpler path is still the S3 media bucket approach — no keys needed, just two `aws s3 cp` commands. Or you could even use SSM `send-command` to push the file content directly.
