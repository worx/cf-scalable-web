### 17:25:46 - Response

GitHub requires a **Personal Access Token** (PAT) instead of a password for HTTPS cloning. You have two options:

**Option 1: Generate a PAT (quick)**
1. Go to: https://github.com/settings/tokens
2. Generate new token (classic) → select `repo` scope
3. Copy the token, then:

```bash
git clone https://YOUR_TOKEN@github.com/worx/cf-scalable-web.git
```

**Option 2: Use SSH key (better long-term)**
```bash
ssh-keygen -t ed25519 -C "deploy-host"
cat ~/.ssh/id_ed25519.pub
```
Copy the public key, add it at https://github.com/settings/keys, then:
```bash
git clone git@github.com:worx/cf-scalable-web.git
```

Also note — the repo might be `cf-scalable-web` not `cf-scalable-drupal` on GitHub (check the remote name). You can verify from your laptop:

```bash
git remote -v
```

The SSH approach is better for the deploy host since there are no credentials to expire (PATs have expiration dates).
