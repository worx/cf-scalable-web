### 16:53:09 - Response

Pushed as `2ec065e`.

**For your current situation on the deploy-host:**

```bash
git pull origin main
make ssm-migration-jumpbox
```

Now you'll get a useful diagnosis instead of "jumpbox not deployed." The most likely first message will be:

```
session-manager-plugin not installed.
On the deploy-host (Ubuntu ARM64):
  sudo curl -sSL "https://s3.amazonaws.com/session-manager-downloads/plugin/latest/ubuntu_arm64/session-manager-plugin.deb" -o /tmp/smp.deb
  sudo dpkg -i /tmp/smp.deb && rm /tmp/smp.deb
```

After that, the second pre-check will probably hit:

```
ZoningInfoAdmin profile not configured (or credentials invalid/expired).
Set up per docs/MIGRATION.md step 4 — copy the [ZoningInfoAdmin] block from
your ~/.aws/config and ~/.aws/credentials onto the host running 'make'.
```

Both are step-4 items. Once they're done, `make ssm-migration-jumpbox` opens an interactive shell, and `make ssm-migration-tunnel` opens the port-forwarding tunnel for pgloader.

**The tunnel target** is what you'll spend most of your time in. The full workflow on deploy-host now is:

```bash
# tmux window 1: open the tunnel
make ssm-migration-tunnel
# Leaves the foreground busy with "Waiting for connections..."

# tmux window 2: run pgloader against localhost:3306
source /etc/worxco/envs/sandbox
# ... write /tmp/zinew.load ...
pgloader /tmp/zinew.load
```

Standing by for the next finding (or success).
