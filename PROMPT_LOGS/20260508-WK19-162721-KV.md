**Architecture decision before I edit:** PHP-FPM passes env vars to workers via `env[KEY] = $VALUE` lines in the **pool config**, not `php.ini`. So the boot script needs to write a snippet into `/etc/php/8.3/fpm/pool.d/www.conf` (or a dedicated `/etc/php/8.3/fpm/pool.d/drupal.conf` overlay) **before** starting PHP-FPM.

Two patterns to choose from:
- **A) Append-to-www.conf** — simpler; one `env[...]` block written between `# BEGIN/END worxco-drupal` markers, idempotent on reboot.
- **B) Drop-in pool config** — write `pool.d/drupal-env.conf` containing only the `env[...]` lines (PHP-FPM merges all `pool.d/*.conf` for the pool). Cleaner separation, easier to drop in or remove.

I'd lean toward **B** — the env-var block is logically separate from the pool's process-management config, and it's easy to regenerate without touching the canonical `www.conf`.

Want me to proceed with B and edit the canonical source in `cloudformation/cf-image-builder.yaml`? (And keep the reference copy in `image-builder/components/install-php-fpm-83.yaml` in sync, plus the same change for PHP 7.4.)
