### 15:44:29 - Response

**Found it. Two stacked bugs:**

1. **IAM stack was never re-deployed** after we added `worxco/sandbox/drupal/*` to PHPInstanceRole last week (commit 86a1fe5). The role's live policy still has only `ssh-keys/*, root-password*, rds/*, cache/*` — **drupal/* is missing**.

2. **The IAM stack was last `CREATE_COMPLETE` on 2026-05-08T15:21** (initial creation, never updated since). The template change we committed is sitting in git but never made it to AWS.

So at boot every PHP box runs `configure-php.sh`, tries to read `worxco/sandbox/drupal/db-password`, gets AccessDenied (with retries — that's why the SSM probe hangs now), the `|| echo ""` fallback substitutes empty, the marker block gets written with `env[DRUPAL_DB_PASS] = ` empty, and PHP-FPM workers serving requests handle the empty value by closing the connection mid-response → nginx 502.

**Fix is one Make target + restart PHP-FPM** (no AMI rebuild, no instance refresh):

```bash
# On deploy-host or your Mac
cd ~/projects/cf-scalable-web   # or wherever
make deploy-iam ENV=sandbox

# Verify the live policy now has drupal/*
aws iam get-role-policy --role-name sandbox-php-instance-role \
  --policy-name SecretsManagerAccess \
  --query 'PolicyDocument.Statement[0].Resource' --output text | tr ',' '\n' | grep drupal

# IAM role changes propagate automatically — but configure-php.sh only runs at
# boot, so the existing www.conf still has the empty password. Re-run the
# boot script on each PHP box, then restart php-fpm so it re-reads the file.
for ASG in sandbox-php74-asg sandbox-php83-asg; do
  for I in $(aws autoscaling describe-auto-scaling-groups \
      --auto-scaling-group-names "$ASG" \
      --query 'AutoScalingGroups[0].Instances[?LifecycleState==`InService`].InstanceId' \
      --output text); do
    echo "$ASG / $I — re-running configure-php.sh + restart fpm"
    aws ssm send-command --instance-ids "$I" \
      --document-name "AWS-RunShellScript" \
      --parameters '{"commands":["sudo /opt/worxco/configure-php.sh","sudo systemctl restart php*-fpm","sudo php-fpm$(cat /opt/worxco/php-version) -tt 2>&1 | tail -3"]}' \
      --query 'Command.CommandId' --output text
  done
done
```

After that, re-curl `/_test.php` and you should get `PHP_OK` + all the env vars filled + `PG_OK`. Then the original test (a real Drupal page) should also work.
