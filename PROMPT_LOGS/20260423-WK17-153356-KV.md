### 15:35:00 - Response

OK, here's the full picture. The workflow is:

1. **Image Builder creates AMIs** (via `make build-amis`)
2. **You manually run `make update-ami-param`** for each pipeline to write the AMI IDs to SSM
3. **Compute stacks read those SSM parameters** at deploy time

Since we tore everything down, there are no AMIs and no SSM parameters. You need to:

```bash
# 1. Trigger all three Image Builder pipelines
make build-amis ENV=sandbox

# 2. Wait for them to complete (~20-30 min each)
#    Check status with: make verify-image-builder ENV=sandbox

# 3. Once complete, write AMI IDs to SSM
make update-ami-param ENV=sandbox PIPELINE=nginx
make update-ami-param ENV=sandbox PIPELINE=php74
make update-ami-param ENV=sandbox PIPELINE=php83

# 4. Then retry compute deployment
make deploy-compute ENV=sandbox
```

This is also a gap in `deploy-all` — it doesn't trigger Image Builder builds or populate AMI parameters before attempting compute deployment. That's probably something to add to the dependency resolver TODO. For now, the builds need to be triggered and completed manually before compute can deploy.
