### 17:59:26 - Response

You're 100% right about excluding `sites/default/files` — clients being clients, filenames in an uploads directory are a hostile environment for shell globs. Should have excluded it from the start. Let me fix both things.

## Better find/chmod pattern (excluding client uploads)

```bash
sudo find /var/www/drupal \
    -path '*/web/sites/default/files' -prune -o \
    -type f -print0 \
  | sudo xargs -0 -I{} grep -l '^#!' {} 2>/dev/null \
  | sudo xargs -d '\n' chmod +x
```

Key changes:
- `-path '*/web/sites/default/files' -prune` — skips the client files tree entirely (`-prune` says "don't descend into this")
- `-print0` + `xargs -0` — null-terminated filenames, so spaces don't break things
- `xargs -d '\n'` on the outer — grep's output is one path per line

Even cleaner alternative using find's own `-exec`:

```bash
sudo find /var/www/drupal \
    -path '*/web/sites/default/files' -prune -o \
    -type f -exec sh -c 'head -c2 "$1" | grep -q "^#!" && chmod +x "$1"' _ {} \;
```

Slower (spawns a shell per file) but handles filenames with any character. For a one-time fix on ~10K vendor files, still finishes in a few seconds.
