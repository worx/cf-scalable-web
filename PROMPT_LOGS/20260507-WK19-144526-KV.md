### 14:46:27 - Response

My bad — I made that up. There's no `user:list` in standard drush. Drush has commands for managing individual users but no built-in "list all users" — historically that's done via `sql:query`.

Here's what actually works:

```bash
# Info about a specific user
vendor/bin/drush user:information admin
vendor/bin/drush user:information --uid=1

# All users via SQL
vendor/bin/drush sql:query "SELECT uid, name, mail, status FROM users_field_data WHERE uid > 0;"

# Discover all available drush commands
vendor/bin/drush list

# Drush commands grouped by namespace
vendor/bin/drush list user
vendor/bin/drush list config
vendor/bin/drush list state
```

The `drush list user` view will show you the full set of user-related commands available — much more reliable than trusting my memory. 😅

For our **install validation**, the simplest authoritative checks are:

```bash
vendor/bin/drush status                                                                  # bootstrap, version, db driver
vendor/bin/drush user:information admin                                                  # admin user exists
vendor/bin/drush sql:query "SELECT count(*) FROM node_field_data;"                       # node table exists
vendor/bin/drush state:get system.cron_last                                              # state subsystem works
```

If all four return without error, the install is healthy.

To answer your question about recent drush changes: this command (`user:list`) didn't exist in any recent version I can recall — drush has historically left bulk user listing to SQL or contrib modules. So my earlier message was just wrong, not version skew. Apologies.
