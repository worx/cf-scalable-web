### 11:55:09 - Response

Two layers here.

**Mac keyboard shortcut (no config needed):** `fn + ↑` = PageUp, `fn + ↓` = PageDown. Same for `fn + ←/→` = Home/End. That works right now.

**Why `Ctrl-u/d` and `?` didn't work:** tmux's copy-mode defaults to **emacs** key bindings on most distros, but you were trying **vi** bindings. In emacs mode, `Ctrl-r` is reverse search, not `?`. To switch the deploy-host's tmux to vi bindings:

```bash
echo 'set -g mode-keys vi' >> ~/.tmux.conf
tmux source-file ~/.tmux.conf   # reload in the running session
```

After that you get the vi set inside copy-mode: `Ctrl-u`/`Ctrl-d` half-page, `Ctrl-b`/`Ctrl-f` full page, `/` forward search, `?` reverse search, `n`/`N` next/prev match, `g`/`G` top/bottom, `0`/`$` line start/end.

**Persistence caveat:** The deploy-host's `~/.tmux.conf` lives on the root EBS volume — when the auto-scaling group rotates the deploy-host instance (or you `make destroy-deploy-host && make deploy-deploy-host`), it's gone. If you want this baked in, two options:

1. Quick: drop `set -g mode-keys vi` into `/etc/tmux.conf` (system-wide) on the deploy-host AMI via the Image Builder component.
2. Better: add to the deploy-host bootstrap script that sets up the ubuntu user's dotfiles. Since deploy-host doesn't get rebuilt often, manual is fine for now — just remember to re-run after a rebuild.

For the 55" screen, you might also add `set -g history-limit 50000` (default is 2000 — way too small for the kind of make output you're generating) and `set -g mouse on` to scroll with the trackpad in non-copy-mode. Want me to write a `.tmux.conf` worth keeping?
