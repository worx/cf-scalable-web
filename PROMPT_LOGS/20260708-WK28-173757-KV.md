## The 54 vs 413 mystery

**Very likely cause:** pgloader's default behavior is to create tables in a schema **named after the source database** — so your 413 tables probably landed in a schema called `zinew` (or similar), not in `public`. My earlier `table_schema='public'` filter would then only match the ~54 tables left over from the `install-drupal-full` fresh Drupal install (which pgloader left alone because they weren't in the source MySQL).

Verify with:

```bash
PGPASSWORD="$DRUPAL_DB_PASS" psql \
  "host=$DRUPAL_DB_HOST port=$DRUPAL_DB_PORT dbname=$DRUPAL_DB_NAME user=$DRUPAL_DB_USER sslmode=require" \
  -c "SELECT table_schema, COUNT(*) AS n_tables
        FROM information_schema.tables
       WHERE table_type='BASE TABLE'
       GROUP BY table_schema
       ORDER BY n_tables DESC;"
```

That will show something like:

```
 table_schema | n_tables
--------------+----------
 zinew        |      413    ← your migrated data
 public       |       54    ← Drupal install leftovers
```

If that's the shape, **the migration worked correctly** — the data is just in a schema Drupal doesn't know to look at. We fix that in `settings.php` (below).

## About COUNT(*) and ANALYZE in PostgreSQL

**Yes, `SELECT COUNT(*)` in PostgreSQL always does a sequential scan** for exact counts. Unlike MySQL InnoDB (which also scans, but often caches recently), PostgreSQL doesn't cache row counts — MVCC makes it hard.

For fast **approximate** counts (accurate to ~99% after ANALYZE runs), use:

```sql
SELECT relname, n_live_tup
  FROM pg_stat_user_tables
 ORDER BY n_live_tup DESC;
```

That's from PostgreSQL's stats collector — updated by autovacuum + your manual ANALYZE. It's what my earlier query used.

## How to run ANALYZE (Postgres newbie edition)

**The command:**

```sql
ANALYZE;              -- Analyzes ALL tables in the current database
ANALYZE zi_zu_link_rev;  -- Analyzes one specific table
```

**How to run it from the command line:**

```bash
PGPASSWORD="$DRUPAL_DB_PASS" psql \
  "host=$DRUPAL_DB_HOST port=$DRUPAL_DB_PORT dbname=$DRUPAL_DB_NAME user=$DRUPAL_DB_USER sslmode=require" \
  -c "ANALYZE;"
```

**Breaking down each piece for you:**

| Piece | What it does |
|-------|--------------|
| `PGPASSWORD="..."` | Sets the password in the environment (avoids interactive prompt) |
| `psql` | The PostgreSQL command-line client (like `mysql` for MySQL) |
| `"host=... dbname=... ..."` | Connection string. Once you say `dbname=drupal`, psql connects to that DB and everything after is scoped to it. You do NOT need to say "analyze database drupal" — being connected to it is enough |
| `-c "ANALYZE;"` | Runs the SQL you pass and exits. Like `mysql -e "..."` |

**How long ANALYZE takes:** for a 12M-row DB it's probably 2-5 minutes. It's mostly I/O — samples ~30K random rows per table to update statistics. Not a full scan.

**When to run it:** now, after the pgloader migration finished. And also after any big data load or schema change in the future.
