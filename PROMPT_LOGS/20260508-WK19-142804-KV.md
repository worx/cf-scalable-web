### 14:29:06 - Response

Two likely causes, both worth checking. Most likely: **the new deploy-host doesn't have your public key on it** — fresh instance, fresh `~/.ssh/authorized_keys`, our bootstrap doesn't auto-load user keys yet (that's the open TODO from a few days ago).

When SSH can't auth with the keys it tries, it eventually falls through to ANY remaining encrypted key in your agent and prompts for the passphrase to try that one too. Hence the "passphrase for a key I'm not using" symptom — SSH is desperately offering everything because nothing's working.

## Step 1 — verify and fix the missing key

SSM directly to the new instance (bypasses SSH entirely) and add your key:

```bash
# On your Mac
INSTANCE_ID=$(AWS_PROFILE=ZI-Sandbox aws ec2 describe-instances \
  --filters "Name=tag:Name,Values=cf-deploy-host" "Name=instance-state-name,Values=running" \
  --query 'Reservations[].Instances[].InstanceId' --output text \
  --region us-east-1)
echo "$INSTANCE_ID"

AWS_PROFILE=ZI-Sandbox aws ssm start-session --target "$INSTANCE_ID" --region us-east-1
```

Once you're in (as `ubuntu`):

```bash
mkdir -p ~/.ssh && chmod 700 ~/.ssh
cat >> ~/.ssh/authorized_keys <<'EOF'
<paste your public key here, e.g., ssh-ed25519 AAAAC3... user@mac>
EOF
chmod 600 ~/.ssh/authorized_keys
exit
```

Get your key from your Mac with `cat ~/.ssh/id_ed25519.pub` (or whichever you use).

Then back on Mac, try `ssh deploy-host` — should work now.

## Step 2 — if it STILL prompts for the wrong key

That's the SSH config side. Two things to check:

```bash
# Show which keys SSH actually offers and which Host blocks match
ssh -v deploy-host 2>&1 | grep -E "Reading config|debug1: (Connecting|Host|Will|Offering|Authentications)" | head -30
```

Most likely fix — add `IdentitiesOnly yes` to your Host block so SSH only offers the explicitly-listed key:

```
Host deploy-host sandbox
  HostName i-NEWINSTANCEID
  User ubuntu
  IdentityFile ~/.ssh/id_ed25519
  IdentitiesOnly yes                  # ← add this
  StrictHostKeyChecking accept-new
  ProxyCommand sh -c "AWS_PROFILE=ZI-Sandbox aws ssm start-session --target %h --document-name AWS-StartSSHSession --parameters 'portNumber=%p'"
```

Without `IdentitiesOnly yes`, SSH offers every key in your agent in order, and if any are passphrase-protected, you get prompted for them. With `IdentitiesOnly yes`, only the listed key is offered.
