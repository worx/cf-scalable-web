### 09:54:44 - Response

The fstab line won't work as-is — it has `{fsx_dns}` as a placeholder, not the actual DNS name. The boot script was supposed to mount it dynamically using the SSM parameter, not through fstab.

Try this on the instance:

```bash
FSX_DNS=$(aws ssm get-parameter --name "/sandbox/fsx/dns-name" --query 'Parameter.Value' --output text)
echo $FSX_DNS
sudo mount -t nfs4 -o nfsvers=4.1,rsize=1048576,wsize=1048576,hard,timeo=600,retrans=2,noresvport,noatime "$FSX_DNS:/fsx" /var/www
```

If the mount command hangs or fails, that's the network/security group issue to debug. If it succeeds, then the boot script just needs fixing to work properly.
