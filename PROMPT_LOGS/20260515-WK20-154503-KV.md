## What to do on the running deploy-host to pick this up

This needs a sequence — `use-env` doesn't exist yet on the live box, and `/var/www` currently holds the `/var/www/sandbox/` legacy mount. Run on the deploy-host:

```bash
# 1. Pull the new code
cd ~/projects/cf-scalable-web
git pull

# 2. cd somewhere safe before touching /var/www
cd ~

# 3. Re-run bootstrap.sh — installs use-env, updates zshrc with the
#    use-env wrapper, removes the old mount-env, but does NOT auto-mount
#    anything because /etc/worxco/current-env is "NONE"
sudo bash scripts/deploy-host/bootstrap.sh

# 4. The legacy mount at /var/www/sandbox is still there.
#    Unmount + remove its fstab entry:
sudo umount /var/www/sandbox
sudo sed -i.bak '/^# worxco-mount-env:/,+1d' /etc/fstab && sudo rm -f /etc/fstab.bak
sudo rmdir /var/www/sandbox    # cleanup empty mount point

# 5. Drop into a fresh zsh so the new use-env wrapper is loaded
exec zsh -l

# 6. Switch to sandbox via the new command
use-env sandbox

# 7. Verify
mount | grep /var/www         # FSx is mounted at /var/www
ls /var/www/drupal/web        # see Drupal codebase at the new canonical path
echo $DRUPAL_DB_HOST          # env vars sourced into the shell automatically
```
