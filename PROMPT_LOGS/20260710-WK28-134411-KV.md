## On the make help navigability question

You're describing exactly what mature Makefile-based projects do — **a two-tier help system**. Common pattern:

- `make help` → shows just category names + one-line blurbs (~20-25 lines)
- `make help-<category>` → shows just that category's targets (like `make help-migration` already does!)
- `make help-all` → the current 400-line firehose (kept for those who want to grep/scroll)

You already have this half-built. The migration help I extracted (`help-migration`) is exactly the pattern applied to one category. We'd just replicate it for the other groups and slim down the top-level `make help`.

Sketch of what `make help` would look like:

```
The Worx Company — cf-scalable-web

$(YELLOW)Full-stack lifecycle:$(NC)
  make deploy-all           — Full parallel deploy (~30 min)
  make destroy-all          — Full parallel destroy (~15 min)
  make help-lifecycle       — Detail: deploy-all-*, destroy-all-*, orchestrator variants

$(YELLOW)Individual stacks:$(NC)
  make help-vpc             — VPC + subnets + IGW
  make help-storage         — S3 + FSx OpenZFS
  make help-database        — RDS PostgreSQL
  make help-cache           — ElastiCache Valkey
  make help-compute         — ALB / NLB / NGINX / PHP-FPM ASGs

$(YELLOW)Application & operations:$(NC)
  make help-drupal          — Drupal install / deploy
  make help-image-builder   — EC2 Image Builder / AMIs
  make help-ssm             — SSM management (audit / remediate / report)
  make help-deploy-host     — Deploy-host lifecycle
  make help-migration       — One-time prod→sandbox migration (cd migration if colocated)

$(YELLOW)Utility & status:$(NC)
  make show-params ENV=...  — Show CFN parameters
  make status ENV=...       — Show stack statuses
  make help-testing         — Test / maintenance targets
  make help-all             — Show everything (current 400-line output)
```

That's ~25 lines. Everything else is one `make help-<X>` away.

**Cost estimate:** Extracting the existing help block into 8-9 sub-targets is straightforward but tedious — the content already exists in the current `make help`, we just chop it into named targets. About **45-60 min** for a careful pass with `make help-*` for each category.

## About the UI aside

Just to name-drop what exists in case you want to poke at it later: `just`, `mask`, and `task` are the modern Makefile-alternative CLIs — they parse a simple task definition file and give you built-in `--list` and `--summary` navigation, tab completion, and colorized listings. `just` in particular is popular and would give you category-based navigation for free. Not suggesting we migrate off Make — just pointing at the landscape if the itch grows. For now, Make + the two-tier help pattern gets us 90% of the way there without changing tools.
