```
feat(nginx): decouple compute health from Drupal install state

Before: the ALB /health endpoint lived only inside drupal.conf (written
to FSx by install-drupal.sh). Freshly-booted nginx instances had no
http server listening on port 80 until install-drupal ran — so every
ALB health check failed, ASG replaced the "unhealthy" instance, and
the deploy was load-bearing on the chicken-and-egg of nginx-needs-
Drupal-which-needs-nginx. Today's deploy-allX run hit exactly that:
2 nginx instances replaced by ELB health failures before install-drupal
finished.

Now: the baseline /etc/nginx/nginx.conf carries a default_server block
that returns 200 on /health and 404 on everything else, with no PHP
dependency, no FSx dependency, no Drupal dependency. nginx is healthy
from second one of boot. The Drupal vhost matches by server_name only
(default_server flag removed) — two default_server declarations on
the same listen port would be an nginx config error.

Also restructures deploy-allX's tail end:
- Phases 1-4 determine deployment success/failure (infrastructure)
- Phase 5 (install-drupal) runs best-effort; failure prints a clear
  "infrastructure is still healthy — re-run with make install-drupal-
  remote ENV=..." and does NOT exit non-zero
- Phase 6 (smoke-test) only runs if Phase 5 succeeded; same best-effort
- Final summary distinguishes "infrastructure healthy" from "Drupal app
  status — see Phase 5/6 output"

This matches the stated design intent: Drupal is an application that
happens to run on the infrastructure, NOT a hard dependency of the
infrastructure deployment. install-drupal is in deploy-allX purely
for operator convenience.

RecipeVersion bumped 1.0.10 → 1.0.11 so build-amis-if-needed will
trigger the AMI rebuild on next deploy-allX. Note: this triggers ALL
3 pipelines (nginx, php74, php83), not just nginx — the version is
shared across recipes. Per-pipeline versioning is a future refactor;
the wasted ~15 min on PHP rebuilds is acceptable for this fix.

Files Modified:
- image-builder/configs/nginx/nginx.conf (added default_server with /health)
- scripts/publish-drupal-vhost.sh (drop default_server + /health block)
- scripts/deploy-host/install-drupal.sh (same drop in inline drupal.conf)
- Makefile (Phases 5/6 best-effort in deploy-allX)
- cloudformation/parameters/image-builder-sandbox.json (RecipeVersion bump)

Tests: bash -n on both scripts; jq empty on JSON; make -n deploy-allX
       parses clean. Real validation needs a fresh AMI bake + deploy.
Docs: inline comments explain the design
```
