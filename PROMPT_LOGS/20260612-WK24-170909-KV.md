fix(docs/migration): use 127.0.0.1 not localhost for tunnel connections

MySQL CLI client treats `-h localhost` as a special token meaning
"skip TCP, use UNIX domain socket at /var/run/mysqld/mysqld.sock"
regardless of any TCP listener on port 3306. Past incident 2026-06-12:
operator (Kurt) opened the SSM port-forwarding tunnel successfully
and tried to verify with `mysql -h localhost -P 3306 ...` per the
runbook — got ERROR 2002 (HY000) "Can't connect to local MySQL server
through socket" every time. Same result with both -p and -P (the
port flag was a red herring; lowercase vs uppercase didn't matter).

Tunnel itself was working fine; the verification command was broken.

Fix:
- Verification command in step 5 changed to `mysql -h 127.0.0.1`
- pgloader load file URL changed to `mysql://...@127.0.0.1:3306/...`
  for consistency (pgloader's MySQL driver doesn't have the
  libmysqlclient localhost-special-case, but using the IP everywhere
  is unambiguous)
- New entry in "Things to watch for" explaining the gotcha and the
  exact error message to recognize ("ERROR 2002 ... socket ...
  mysqld.sock"), so a future operator can map the symptom to the
  cause immediately

Remaining `localhost:3306` references in the architecture diagram
caption + narrative text are fine — those are descriptions, not
copy-paste commands.
