Add zsh to the deploy-host AMI's bootstrap, configure a system-wide

Implementation notes:
- chsh -s /bin/zsh ubuntu sets ubuntu's login shell, but SSM Session
  Manager bypasses /etc/passwd via its own SSM-SessionManagerRunShell
  document (which forces `exec bash -l`). To get zsh in SSM sessions
  without changing the account-level SSM doc (which would affect every
  box we SSM into), append an auto-exec snippet to /etc/bash.bashrc
  that swaps bash → zsh for interactive shells. Escape hatch: prefix
  with NO_AUTO_ZSH=1 to keep bash.

  prompt. Documented in gotchas.md so future-us doesn't hit it again.
- INC_APPEND_HISTORY + SHARE_HISTORY make every command available
  across concurrent shells immediately (vs zsh's default per-shell
  history file at exit).
- HISTSIZE/SAVEHIST = 100000 — generous, deploy-host history is
  intentionally long-running ops record.
- All idempotent: re-running bootstrap.sh appends nothing twice
  (greps before append).

Follow-up for next session:
- `use-env <name>` script that umounts /var/www, remounts the named
  env's FSx, sources /etc/worxco/envs/<env>, writes current-env.
