Yesterday's destroy/redeploy cycle still produced an instance missing
PHP, composer, and MOTD even after the HOME=/root fix — symptoms

Strategic fix: move the toolchain installation out of the UserData

heavy lifting lives in scripts/deploy-host/bootstrap.sh.

Wins:
  * Debuggable: bootstrap.sh is a normal bash script. SSM into the
    box, run sections manually to find failures without another
    full destroy/deploy cycle.
  * Iterable: edit bootstrap.sh, run it, see results in seconds
    instead of 10-minute deploy cycles.
  * Visible: full set -x trace, plus a step() function that prints
    a clear marker + memory + disk snapshot before each section.
    Failure trap reports the failing line and last step name.
  * Smaller UserData: 4.7K (was 26.7K at peak yesterday). No future
    16K-limit surprises.
  * Idempotent: bootstrap.sh re-runs safely. apt installs already
    idempotent; install -m overwrites; .bashrc additions guarded by
    grep checks; composer global require is a no-op if already there.

bootstrap.sh structure:
  - ERR trap reports exit code, line, last step, memory, disk
  - step() function: section header + resource snapshot
  - set -x trace mode for full command visibility in the log
  - Defensive HOME=/root + AWS_DEFAULT_REGION (with IMDS fallback)
  - 16 step() blocks covering apt upgrade, core tools, profile.d,
    AWS credentials README, root password, cfn-lint, Node+Claude,
    SSM agent, Drupal apt packages (NFS, psql, redis, PHP+11 ext),
