### 17:49:59 - Response

`/tmp/zinew.load` — that's the path the doc uses. Two reasons for those choices:

- **`/tmp/` because it's temporary and contains secrets.** The file has the prod MySQL password and the sandbox PostgreSQL password embedded in URLs. `/tmp/` clears on reboot, isn't backed up, and lives outside any source tree. After the run, the doc has you `shred -u /tmp/zinew.load` to wipe it.
- **`.load` because that's pgloader's convention** for its load-file DSL (the `LOAD DATABASE … FROM … INTO … WITH … CAST …` syntax). pgloader auto-recognizes the format and parses accordingly.
- **`zinew` because that's your source DB name** — easy to track what the file is for if you end up with multiple migration attempts. Could just as easily be `prod-import-20260612.load` or whatever your future self would recognize.

The full sequence (with the file path in context) — copy-pasteable:

```bash
source /etc/worxco/envs/sandbox

read -s -p "Prod MySQL password for user 'worxco': " PROD_PW; echo
PROD_PW_ENC=$(printf '%s' "$PROD_PW" | jq -sRr @uri)
SANDBOX_PW_ENC=$(printf '%s' "$DRUPAL_DB_PASS" | jq -sRr @uri)

cat > /tmp/zinew.load <<EOF
LOAD DATABASE
  FROM      mysql://worxco:${PROD_PW_ENC}@127.0.0.1:3306/zinew
  INTO      postgresql://${DRUPAL_DB_USER}:${SANDBOX_PW_ENC}@${DRUPAL_DB_HOST}:${DRUPAL_DB_PORT}/${DRUPAL_DB_NAME}

WITH include drop, create tables, create indexes, reset sequences, foreign keys,
     downcase identifiers, batch rows = 1000, prefetch rows = 1000

CAST type tinyint when (= 1 precision) to boolean drop typemod using tinyint-to-boolean,
     type datetime to timestamptz drop default drop not null using zero-dates-to-null,
     type date drop not null drop default using zero-dates-to-null,
     type timestamp drop default drop not null using zero-dates-to-null
;
EOF
chmod 600 /tmp/zinew.load
unset PROD_PW

pgloader /tmp/zinew.load 2>&1 | tee /tmp/zinew-pgloader.log

# Cleanup
shred -u /tmp/zinew.load
unset PROD_PW_ENC SANDBOX_PW_ENC
```
