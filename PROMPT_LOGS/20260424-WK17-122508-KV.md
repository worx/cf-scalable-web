### 12:25:23 - Response

Correct — the PHP instances are in private subnets with no NAT gateway. They can only reach AWS services via the VPC endpoints we set up (SSM, Secrets Manager, S3). General internet (getcomposer.org) is unreachable.

You have a few options:

1. **Install Composer from the deploy host** — download it there (has internet), then push it to S3, then pull from S3 on the PHP instance
2. **Add Composer to the Image Builder AMI** — bake it in so every instance has it
3. **Add a NAT gateway** — gives private instances full internet access (~$32/mo)

Option 1 is quickest for right now. From the deploy host:

```bash
curl -sS https://getcomposer.org/installer -o /tmp/composer-setup.php
aws s3 cp /tmp/composer-setup.php s3://sandbox-image-builder-kv-worxco/tools/composer-setup.php
```

Then on the PHP instance:

```bash
aws s3 cp s3://sandbox-image-builder-kv-worxco/tools/composer-setup.php /tmp/composer-setup.php
php /tmp/composer-setup.php --install-dir=/usr/local/bin --filename=composer
```

The S3 gateway endpoint means the PHP instance can reach S3 without internet.
