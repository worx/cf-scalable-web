## Cloud install workflow

Once you have a deployed environment (`make deploy-allX ENV=sandbox`), this is the path:

```bash
ssh deploy-host
cd ~/projects/cf-scalable-web && git pull

# One-time per session: mount FSx + cache /etc/hosts entry
sudo mount-env sandbox
sudo refresh-env-config sandbox

# Install
make install-drupal ENV=sandbox     # ~3-5 min

# Verify
make verify-drupal ENV=sandbox

# Browse via drush rs + tunnel (works until PHP-FPM boot script is wired)
source /etc/worxco/envs/sandbox
export DRUPAL_DB_PASS="$(aws secretsmanager get-secret-value \
  --secret-id worxco/sandbox/drupal/db-password --query SecretString --output text)"
cd /var/www/sandbox/drupal && vendor/bin/drush runserver 0.0.0.0:8080

# Mac: ssh -L 8080:localhost:8080 deploy-host  →  http://localhost:8080
```

Login: `admin` + the password from `aws secretsmanager get-secret-value --secret-id worxco/sandbox/drupal/admin-password --query SecretString --output text`

## Key design decisions baked in

| Decision | Why |
|----------|-----|
| Auto-create Secrets Manager entries on first run | So script works standalone before cf-app-drupal.yaml exists. The CFN stack (step 3) will own them with proper retention. |
| `hash_salt` in `/var/www/<env>/drupal-private/salt.txt` | Persists across reinstalls — sessions don't die when codebase is rebuilt. |
| `drupal_user` owns the database | Lets drush site:install run without superuser; minimum privilege Drupal can operate with. |
| `settings.php` is `getenv()`-driven | Same file works on deploy-host (with our exports) AND on PHP-FPM (with boot script env vars). One source of truth. |
| Idempotent CREATE-or-ALTER on the user | If you re-run install (after remove), password rotation Just Works. |
| Doesn't auto-mount FSx | Keeps "is FSx mounted" as a clearly visible explicit prerequisite. Less magic, easier debugging. |
| Production guard | `ENV=production` requires typing the env name (install) or "WIPE PRODUCTION" (remove). Cheap insurance. |

## What's still missing (next: step 3 — `cf-app-drupal.yaml`)
