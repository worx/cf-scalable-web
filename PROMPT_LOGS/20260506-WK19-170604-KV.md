## Things to Watch on Review

A few choices in the plan I made that you may want to push back on:

1. **Per-env FSx subdirectories** (`/var/www/sandbox/`, `/var/www/staging/`) on deploy-host. Alternative is mounting one env at a time. I went with per-env subdirs because it's simpler and you'd typically want to inspect multiple envs.

2. **Drush 13** (whatever ships with Drupal 11) — installed via `composer require drush/drush` in the project, not globally. This means each Drupal install has its own Drush. Standard practice.

3. **`make drush -- <args>` syntax** — I sketched `$(filter-out $@,$(MAKECMDGOALS))` which is a known-clunky Make pattern. Alternative: `make drush ARGS="cr"` is uglier but more reliable. We can decide later when actually building.

4. **TrustedHostPatterns** in settings.php includes a generic `^.+\.elb\.amazonaws\.com$` to allow any ALB DNS. Less strict but more flexible. Could tighten to specific ALB DNS.

5. **No backup of installed sites** — explicitly out of scope. If you want backup story for v1, let me know.

Read through it and let me know what to adjust. The plan file is in git so we can iterate without losing work.
