## Time_Billing Character Handling

**Python side (creator)** — `sanitize_unicode()` at line 304 of `extract_pending_notes.py`:
- Replaces smart quotes (`\u201c`, `\u201d`) → regular quotes
- Replaces em dash (`\u2014`) → `--`, en dash (`\u2013`) → `-`
- Replaces ellipsis (`\u2026`) → `...`
- Reason: Windows doesn't render `\uxxxx` escapes properly in the AHK consumer

**AHK side (consumer)** — `TimeBillingAutomation.ahk`:
- Line 167-168: Unescapes JSON `\n` → real newlines, `\"` → real quotes
- Line 267: Converts Unix `\n` → Windows `\r\n` before pasting via clipboard

**For us**: Since we're generating text directly (not parsing Apple Notes HTML), we won't get smart quotes. But we should apply the same `sanitize_unicode` treatment as a safety net — Claude occasionally uses em dashes in natural language. Easy to handle.

### Hook: `time-tracker.sh` (global, `~/.claude/hooks/`)

Fires on `UserPromptSubmit` and `Stop`. Logic:

**On UserPromptSubmit:**
1. Check if `.claude/timekeeping.json` exists in `$CLAUDE_PROJECT_DIR` — if not, exit
2. Read `clock.json` state
3. If no state or different date → auto-close previous day (if unfinalized), start fresh block
4. If `last_stop` was <30 min ago → extend current block (do nothing, it's still open)
5. If `last_stop` was >30 min ago → close previous block at `last_stop + 10min`, start new block
6. Record the current prompt log filename for later summarization

**On Stop:**
1. Update `last_stop` timestamp in `clock.json`

### Skill: `/timeclose`

1. Read `clock.json` for today (or specified date)
2. Close any open block (end = now + 10min, or now if explicitly closing)
3. Read all referenced prompt logs
4. Generate bullet-point summaries per time block (this is where Claude's brain shines)
5. Write finalized `sheets/YYYY-MM-DD.json`
6. Clear `clock.json`

### Skill: `/timepost`

2. For each, format into the `pending_billing.json` structure
3. **Overwrite** `~/Downloads/pending_billing.json`
4. Mark sheets as posted
5. Report what was written
