fix(install-drupal): psql -v + URL-encode password + operational ordering note

Three changes to install-drupal.sh, motivated by the 2026-05-15 debug arc:

1. Replace the un-quoted-heredoc + DO block with psql `-v` substitution
   and single-quoted heredocs. The original form actually wasn't buggy
   (bash variable substitution is single-pass — backticks and `$` chars
   IN the substituted value don't get re-evaluated), but the psql form
   is cleaner, eliminates the `\$\$` escaping noise from PL/pgSQL DO
   blocks, and is the right pattern to copy elsewhere. `:'var'` and
   `:"var"` handle proper escaping of single/double quotes in values.

2. URL-encode the password before embedding in drush --db-url. The
   cf-app-drupal ExcludeCharacters set permits `,` (RFC 3986 says OK in
   userinfo) but various URL parsers in the wild are picky. python3 is
   already in the deploy-host's bootstrap toolchain, so the encoding is
   a one-liner.

3. Add an operational-ordering comment near the password-resolution
   block: after any cf-app-drupal deploy that may have regenerated the
   GenerateSecretString secret value, install-drupal.sh MUST be re-run
   so the postgres user gets re-ALTERed to match. This is the actual
   root cause of the 5-15 password mismatch — not a code bug, but an
   undocumented operational requirement now spelled out.
