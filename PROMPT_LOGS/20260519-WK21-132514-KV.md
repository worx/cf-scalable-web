### 13:26:00 - Response

Two clean answers:

**1. Idempotency of Drupal stack (Phase 6) on re-run:** Yes, fully idempotent. The `cf-app-drupal` stack survived `destroy-all` (it's a separate stack), so when Phase 6 runs again it's a no-op CFN update (empty changeset). The key bit: `GenerateSecretString` only generates on CREATE, never on UPDATE — so the existing secret values are preserved, not rotated. Phase 7's install-drupal.sh will read those preserved values and re-create the postgres user in the brand-new RDS with the same password (ALTER USER / CREATE USER pattern).

So the three "pieces" of the Drupal install end up like this:

| Piece | After destroy-all yesterday | After deploy-allX today |
|---|---|---|
| `cf-app-drupal` stack (secrets + SSM params) | Preserved | No-op redeploy (Phase 6) |
| RDS drupal database + schema | Gone (RDS destroyed) | Rebuilt in Phase 7 (drush site:install) |
| FSx codebase + .installed marker | Gone (FSx destroyed) | Rebuilt in Phase 7 (composer create-project) |

**2. Impact of destroying deploy-host on Drupal stuff:** None. `cf-deploy-host` and `cf-app-drupal` are completely separate stacks; destroying the deploy-host VM doesn't touch the Drupal secrets, SSM params, or any AWS resources owned by `cf-app-drupal`. The deploy-host is just a working machine that *runs* install-drupal.sh — it doesn't *hold* any Drupal state.

The only deploy-host-tied secret in play is the GitHub SSH key (`worxco/deploy-host/github-ssh-key` or similar) which `cf-deploy-host` owns. When you redeploy the deploy-host, it reads the same key and clones the repo. Drupal-side: untouched.

One small nuance worth knowing: install-drupal.sh has an explicit "if cf-app-drupal stack got redeployed, re-run me so postgres password stays in sync with the rotated secret" comment. That's a guard for the case where GenerateSecretString *did* rotate. In our case it didn't (no-op update), so the guard isn't load-bearing today — but it's why Phase 7 is built on the assumption that "secrets exist, sync postgres to whatever they say."
