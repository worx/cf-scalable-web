### 15:04:05 - Response

Exactly right — `drupal-sandbox.test` is the only Host header that matches, so curls/browsers hitting the ALB DNS directly fall through to the health-check default server. Mirror what Drupal's `trusted_host_patterns` already accepts (`^.+\.elb\.amazonaws\.com$`) by adding a regex name to nginx:

**Change just the `server_name` line:**

```nginx
server_name drupal-sandbox.test ~^.+\.elb\.amazonaws\.com$;
```

NGINX `server_name` accepts a list of three flavors:
- Exact: `drupal-sandbox.test`
- Wildcard: `*.example.com` (single label only — won't work here because ALB names are `sandbox-alb-xxx.us-east-1.elb.amazonaws.com`, multiple labels deep)
- **Regex** (prefix with `~`): `~^.+\.elb\.amazonaws\.com$` — matches *any* AWS ELB DNS

Two ways to apply on the deploy-host:

**A) Just re-paste the whole heredoc** with the line updated (works either way — the `tee` overwrites).

**B) In-place edit + reload:**
```bash
sudo sed -i \
  's|^  server_name drupal-sandbox.test;$|  server_name drupal-sandbox.test ~^.+\\.elb\\.amazonaws\\.com$;|' \
  /var/www/nginx/sites-enabled/drupal.conf

# Validate + propagate to every nginx box
for I in $(aws ec2 describe-instances \
    --filters "Name=tag:aws:autoscaling:groupName,Values=sandbox-nginx-asg" \
              "Name=instance-state-name,Values=running" \
    --query 'Reservations[*].Instances[*].InstanceId' --output text); do
  echo "--- $I ---"
  aws ssm send-command --instance-ids "$I" \
    --document-name "AWS-RunShellScript" \
    --parameters 'commands=["sudo nginx -t && sudo nginx -s reload && echo RELOADED"]' \
    --query 'Command.CommandId' --output text
done
```

After that, both `curl -H "Host: drupal-sandbox.test" http://$ALB/` AND `curl http://$ALB/` (Host header defaults to the ALB DNS, regex-matches) should both reach Drupal. Browser test against the raw ALB URL works without any `/etc/hosts` edit.

**Heads-up:** the regex `~^.+\.elb\.amazonaws\.com$` matches *every* ELB in the universe, not just yours. That's fine for sandbox where access is private-VPC anyway, but for prod you'd want to tighten to your specific ALB DNS or just rely on `drupal-sandbox.test` (or your real domain) and skip the regex.
