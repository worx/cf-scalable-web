## Implication for `_common.sh`

Since the shell scripts on jumpbox and deploy-host run outside Make, they need an equivalent that reads the **env var** `CONFIRMED=yes`. The Makefile targets can pass it through:

```make
migration-restore-mysql:
	@CONFIRMED=$(CONFIRMED) ./scripts/migration/deploy-host/restore-mysql.sh
```

And `_common.sh` provides the mirroring function:

```bash
confirm_or_exit() {
  local msg="$1"
  if [ "${CONFIRMED:-}" = "yes" ]; then
    return 0
  fi
  echo -e "\033[0;31mWARNING: ${msg}\033[0m"
  echo -e "\033[0;36m(Set CONFIRMED=yes to skip this prompt for unattended runs.)\033[0m"
  read -p "Type 'yes' to confirm: " confirm
  [ "$confirm" = "yes" ] || { echo "Cancelled"; exit 0; }
}
```

Behaviorally identical to the Makefile prompt — same phrase, same env var, same colors.
