### 14:41:59 - Response

Exactly right. The `-remote` suffix is "dispatch this to a remote host via SSM, from wherever you are." Both these targets are designed to run from your Mac:

- `make install-drupal-remote ENV=sandbox` — runs from Mac, uses AWS CLI to find the deploy-host by tag, sends the install via SSM
- `make smoke-test-drupal ENV=sandbox` — runs from Mac, fetches ALB DNS from CFN output and curls it directly

The naming convention in the project:
- `make install-drupal` → has the `[ -f /etc/worxco/deploy-host-marker ]` guard and refuses to run anywhere else
- `make install-drupal-remote` → no such guard, runs anywhere with AWS CLI

So yes, Mac is the correct place.
