**No make target — by design.** When we built this yesterday, you said: "let's go with documentation that can be read and properly interpreted. ... not always true that this is the same for each future user." So `docs/DNS-SETUP.md` is the runbook, not a Makefile target.

**Paste-ready block for when deploy-allX completes:**

```bash
ENV=sandbox
ENVS_ZONE_ID=Z03149452QE09GQX23F48
ALB_DNS=$(aws --profile ZI-Sandbox cloudformation describe-stacks \
  --stack-name "cf-scalable-web-$ENV-compute-alb" \
  --query "Stacks[0].Outputs[?OutputKey=='ALBDnsName'].OutputValue" --output text)
ALB_ZONE=$(aws --profile ZI-Sandbox cloudformation describe-stacks \
  --stack-name "cf-scalable-web-$ENV-compute-alb" \
  --query "Stacks[0].Outputs[?OutputKey=='ALBHostedZoneId'].OutputValue" --output text)
echo "ALB: $ALB_DNS  zone: $ALB_ZONE"

cat > /tmp/env-alias.json <<EOF
{
  "Changes": [{
    "Action": "UPSERT",
    "ResourceRecordSet": {
      "Name": "$ENV.envs.zoning-info.com.",
      "Type": "A",
      "AliasTarget": {
        "DNSName": "$ALB_DNS",
        "HostedZoneId": "$ALB_ZONE",
        "EvaluateTargetHealth": false
      }
    }
  }]
}
EOF
aws --profile ZI-Sandbox route53 change-resource-record-sets \
  --hosted-zone-id "$ENVS_ZONE_ID" \
  --change-batch file:///tmp/env-alias.json
```

`UPSERT` overwrites the existing record (if any) with the new ALB DNS — so even though destroy-all left the old record pointing at a now-defunct ALB, this command updates it in place. No need to delete-then-create.

**Worth offering:** I *could* write a `make publish-dns ENV=...` target that's generic enough to work for any deployment — it'd read the FQDN from the SSM site-name parameter, derive the sub-zone name from it, look up the sub-zone ID by name, and UPSERT. That keeps the operator from copy-pasting CLI calls but doesn't hardcode `envs.zoning-info.com` or `Z03149452QE09GQX23F48` into the project. Different from what we discussed yesterday in that there's no hardcoding, just plumbing.

Not for right now — you're mid-teardown. But flag in your head whether it's worth my doing after the rebuild settles. The paste-ready block above is fine for tonight.
